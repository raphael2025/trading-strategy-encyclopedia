## 一、风险预算的核心思想

### 1. 传统资金管理 vs 风险预算

**传统资金管理**：
- 关注"投多少"
- 决定仓位比例
- 忽略风险贡献

**风险预算（Risk Budgeting）**：
- 关注"承担多少风险"
- 基于风险贡献分配
- 显式管理风险来源

### 2. 关键原则

**原则 1：不要让单一风险源主导**
- 单一风险源（行业、策略、资产）≤ 30%
- 多样化是核心

**原则 2：风险与回报对称**
- 承担更多风险需要更多回报补偿
- 风险调整后收益（Sharpe）才是关键

**原则 3：动态调整**
- 风险随市场变化
- 定期再平衡

**原则 4：黑天鹅保护**
- 尾部风险是真实的
- 不要被平均收益欺骗

## 二、风险度量指标

### 1. 波动率（Volatility）

**定义**：收益率标准差年化

**计算**：
```
σ_annual = σ_daily × √252
```

**局限**：
- 假设正态分布
- 忽略尾部
- 对极端事件反应慢

### 2. VaR（Value at Risk）

**定义**：在给定时段和置信度下，预期的最大损失

**参数**：
- 时间：1 天 / 10 天 / 1 个月
- 置信度：95% / 99% / 99.9%

**计算方法**：

**历史法**：
- 用历史收益率分布
- 取分位数

**参数法**：
- 假设正态分布
- VaR = μ - z × σ

**蒙特卡洛**：
- 模拟多个路径
- 取分位数

**例**：
- 100 万美元组合
- 1 天 95% VaR = 5%
- 含义：1 天内有 5% 概率损失超过 5 万美元

**局限**：
- 不能捕捉"超过 VaR"的损失
- 黑天鹅期间失效

### 3. CVaR / Expected Shortfall

**定义**：超过 VaR 情况下的平均损失

**比 VaR 更保守**：
- 实际"最坏 5% 情况下的平均损失"
- 对尾部敏感

**例**：
- VaR 95% = 5%
- CVaR 95% = 8%
- 含义：最坏 5% 情况下，平均损失 8%

### 4. 最大回撤（Maximum Drawdown）

**定义**：从历史最高点到最低点的最大跌幅

**计算**：
```python
# 假设 equity 是资金曲线
peak = equity.cummax()
drawdown = (equity - peak) / peak
max_dd = drawdown.min()
```

**意义**：
- 反映"痛苦程度"
- 投资者最关注的指标

**局限**：
- 单一数字，忽略过程
- 恢复时间同样重要

### 5. 卡尔马比率（Calmar Ratio）

**定义**：年化收益 / 最大回撤

**评级**：
- > 3：优秀
- > 1：合格
- < 1：危险

### 6. 索提诺比率（Sortino Ratio）

**定义**：超额收益 / 下行波动率

**比 Sharpe 更好**：
- 只惩罚下行波动
- 投资者不在乎上行波动

## 三、风险预算方法

### 1. 风险平价（Risk Parity）

**核心思想**：每个资产/策略贡献等量风险

**计算**：
- 风险贡献 ∝ 权重 × 边际风险
- 求解：所有资产边际风险贡献相等

**优点**：
- 不需要估计收益
- 风险分散好
- Bridgewater All Weather 经典

**局限**：
- 杠杆问题（高风险资产配低权重）
- 假设相关性稳定

**实战调整**：
- 风险预算平衡
- 杠杆调整

**加密应用**：
- BTC、ETH、Solana 风险平价
- + 现金或稳定币
- + 期权或备兑

### 2. 风险预算（Risk Budgeting）

**核心思想**：根据预期收益分配风险

**步骤**：
1. 估计每个策略的预期收益
2. 估计风险（Sharpe）
3. 计算最优风险预算

**公式**：
- 风险预算 ∝ (预期收益 - 无风险) / 风险

**例**：
- 策略 A：预期 20%，波动率 15%
- 策略 B：预期 10%，波动率 8%
- A 的边际收益：20% / 15% = 1.33
- B 的边际收益：10% / 8% = 1.25
- 差异不大，建议接近 50:50

### 3. 凯利公式（Kelly）

**核心思想**：最大化长期增长率

**公式**：
- f* = (p × b - q) / b
- p = 胜率
- q = 1 - p
- b = 盈亏比

**实战**：
- 用 1/4 Kelly 起步
- 避免过度投注
- 凯利分数 vs 凯利百分比

**多策略**：
- 每策略独立凯利
- 总仓位 ≤ 凯利合计
- 注意相关性

### 4. 最大分散（Maximum Diversification）

**目标**：最大化组合分散度

**公式**：
- Diversification Ratio = (∑w × σ_i) / σ_portfolio
- 最大化此比率

**风险**：忽略相关性估计误差

### 5. Black-Litterman

**核心思想**：融合主观观点 + 市场均衡

**步骤**：
1. 起始：市场均衡权重
2. 加入主观观点（信心度）
3. 反推权重

**优点**：
- 解决 Markowitz 对参数敏感
- 适合大资金
- Goldman Sachs 内部使用

**加密应用**：
- 起始权重：BTC 50% / ETH 30% / 稳定币 20%
- 加入观点：BTC ETF 影响 + 减半影响
- 反推权重

## 五、加密特有风险管理

### 1. 波动率管理

**挑战**：BTC 波动率 60-100%，远超传统资产

**应对**：
- **仓位控制**：单资产 ≤ 5-10%
- **动态止损**：用 ATR（Average True Range）
- **波动率目标**：调整杠杆使组合波动率 ≤ 15%/年
- **多资产分散**：不同币种的低相关性

### 2. 链上风险

**风险类型**：
- 智能合约漏洞
- 桥接被黑
- 私钥丢失
- 预言机操纵
- 治理攻击

**应对**：
- 选择审计过的协议
- 多签 + 硬件钱包
- 分散存储
- 避免无限授权

### 3. 交易所风险

**FTX 暴雷案例**：
- 80 亿美元客户资金亏空
- 用户仅获 10-25% 分配

**应对**：
- **不把币放在交易所**（用硬件钱包）
- 分散多个交易所
- 选择受监管的（如 Coinbase）
- 定期提现测试

### 4. 监管风险

**风险**：
- 监管政策变化
- 制裁
- 交易所被关

**应对**：
- 关注政策变化
- 准备应急方案
- 多法域分散

### 5. 流动性风险

**风险**：
- 极端市价冲击
- 山寨币流动性差
- 退出困难

**应对**：
- 持仓限于主流币
- 大额订单拆分
- 紧急退出计划

### 6. 永续合约特殊风险

**风险**：
- 资金费率（funding）
- 强平
- 操纵清算

**应对**：
- 监控 funding
- 控制杠杆（≤ 3x）
- 硬性止损单

## 七、动态再平衡

### 1. 再平衡方法

**定期再平衡**：
- 每月/季度
- 简单稳定
- 忽略市场状态

**波动率触发**：
- 仓位偏离目标 ± 5%
- 立即再平衡

**信号触发**：
- 信号变化（如趋势反转）
- 调整策略权重

### 2. 再平衡成本

**交易成本**：
- 手续费 0.05-0.10% × 2（买+卖）
- 滑点 0.05%
- 总成本 0.15-0.25%

**税收成本**：
- 短期资本利得税
- 加密税务因法域而异

**再平衡频率**：
- 太频繁：成本高
- 太稀少：风险集中
- **实战**：月度或偏离 5% 触发

### 3. 加密再平衡挑战

**24/7 市场**：
- 没有收盘价
- 用 4h/1d K 线作为"日线"

**周末**：
- 流动性下降
- 调整时机谨慎

**减半事件**：
- 长期市场结构变化
- 调整策略权重

## 八、风险监控与告警

### 1. 实时指标

**仓位风险**：
- 实时仓位 vs 风险预算
- 偏离度 > 10% 报警

**市场风险**：
- 实时 VaR
- 实时波动率
- Funding Rate 异常

**集中度**：
- 最大单仓位
- 行业集中度
- 策略集中度

### 2. 告警分级

**P0（紧急）**：
- 累计回撤 > 20%
- 系统崩溃
- 立即处理（5 分钟内）

**P1（重要）**：
- 单日回撤 > 5%
- 累计回撤 > 10%
- API 失败
- 30 分钟内处理

**P2（常规）**：
- 仓位偏离 > 10%
- 监控指标异常
- 当日处理

**P3（信息）**：
- 通知类
- 每日汇总

### 3. Kill Switch

**触发条件**：
- P0 告警
- 累计亏损 > X%
- 手动触发

**操作**：
- 撤销所有挂单
- 平掉所有仓位
- 停止策略运行

**恢复**：
- 人工评估
- 修复问题
- 重新启动（小资金）

## 九、心理与纪律

### 1. 风险预算是约束

**原则**：
- 风险预算一旦设定，**严格遵守**
- 不要因为"看起来安全"而放松
- 永远假设黑天鹅会来

### 2. 亏损后的应对

**10% 回撤后的检查**：
- 是模型失效还是正常波动？
- 是市场结构变化还是策略问题？
- 是否需要调整风险预算？

**20% 回撤后的行动**：
- 减仓 50%
- 重新评估模型
- 修复或退役

**30% 回撤后的选择**：
- 全部平仓
- 完全重新评估
- 重新开始

### 3. 盈利后的纪律

**不要"因为赚了"而加杠杆**：
- 风险预算不应随盈利变化
- 提取部分利润

**不要"因为赚了"而放松标准**：
- 严格的风险管理是成功的关键
- 不是"赚钱的策略都好"

## A.1 Risk Parity 权重（等风险贡献）

```python
"""
Risk Parity: 每资产对组合风险贡献相等
- 不需要预期收益（vs Markowitz）
- 适合：被动型 / 多资产
"""
import numpy as np
import pandas as pd
from scipy.optimize import minimize


def risk_parity_weights(cov: pd.DataFrame) -> dict:
    n = len(cov)

    def risk_contribution(w):
        port_vol = np.sqrt(w @ cov.values @ w)
        marginal = cov.values @ w
        return w * marginal / port_vol

    def loss(w):
        rc = risk_contribution(w)
        return np.sum((rc - rc.mean()) ** 2)

    constraints = [{'type': 'eq', 'fun': lambda w: w.sum() - 1}]
    bounds = [(0.01, 0.5)] * n
    x0 = np.ones(n) / n
    res = minimize(loss, x0, bounds=bounds, constraints=constraints,
                   method='SLSQP', options={'ftol': 1e-10})
    return dict(zip(cov.columns, res.x))
```

## A.2 最大分散化组合 (MDP)

```python
"""
Maximum Diversification: 最大化 (Σ w_i * σ_i) / σ_portfolio
"""
import numpy as np
import pandas as pd
from scipy.optimize import minimize


def max_diversification_weights(vols: pd.Series, corr: pd.DataFrame) -> dict:
    n = len(vols)
    cov = np.outer(vols, vols) * corr.values
    weighted_avg_vol = vols.values

    def neg_div_ratio(w):
        port_vol = np.sqrt(w @ cov @ w)
        return -weighted_avg_vol @ w / port_vol

    constraints = [{'type': 'eq', 'fun': lambda w: w.sum() - 1}]
    bounds = [(0, 1)] * n
    x0 = np.ones(n) / n
    res = minimize(neg_div_ratio, x0, bounds=bounds, constraints=constraints,
                   method='SLSQP')
    return dict(zip(vols.index, res.x))
```

## A.3 Kelly 仓位

```python
"""
Kelly Criterion: f* = (p/a) - q/b
- 加密建议 Half-Kelly 或 Quarter-Kelly
"""
import numpy as np
import pandas as pd


def kelly_single(win_rate: float, win_loss_ratio: float) -> float:
    p, q = win_rate, 1 - win_rate
    b = win_loss_ratio
    f = p - q / b
    return max(0, f)


def kelly_multi(returns: pd.DataFrame, use_half: bool = True) -> pd.Series:
    mu = returns.mean()
    cov = returns.cov()
    kelly = np.linalg.inv(cov) @ mu
    if use_half:
        kelly = kelly / 2
    return pd.Series(kelly, index=returns.columns).clip(lower=0)
```

## A.4 Black-Litterman 视图整合

```python
"""
Black-Litterman: 融合市场均衡收益 + 投资者主观视图
"""
import numpy as np
import pandas as pd


def black_litterman(pi: pd.Series, cov: pd.DataFrame,
                    P: np.ndarray, Q: np.ndarray,
                    omega: np.ndarray, tau: float = 0.05) -> pd.Series:
    inv_tau_cov = np.linalg.inv(tau * cov)
    inv_omega = np.linalg.inv(omega)
    M = inv_tau_cov + P.T @ inv_omega @ P
    posterior = np.linalg.inv(M) @ (inv_tau_cov @ pi + P.T @ inv_omega @ Q)
    return pd.Series(posterior, index=pi.index)
```

## A.5 风险预算归因

```python
"""
归因每个策略 / 资产对总风险的贡献
"""
import numpy as np
import pandas as pd


def risk_contribution(weights: pd.Series, cov: pd.DataFrame) -> pd.Series:
    port_vol = np.sqrt(weights.values @ cov.values @ weights.values)
    marginal = cov.values @ weights.values
    rc = weights.values * marginal / port_vol
    return pd.Series(rc, index=weights.index)


def risk_budget_check(weights: pd.Series, cov: pd.DataFrame,
                       budget: pd.Series) -> dict:
    rc = risk_contribution(weights, cov)
    actual_pct = rc / rc.sum()
    return {
        'actual_pct': actual_pct,
        'budget_pct': budget / budget.sum(),
        'deviation': (actual_pct - budget / budget.sum()).abs().max()
    }
```

## A.6 动态再平衡

```python
"""
定期再平衡（threshold-based / time-based）
"""
import pandas as pd


class Rebalancer:
    def __init__(self, target_weights: pd.Series, threshold: float = 0.05):
        self.target = target_weights
        self.threshold = threshold
        self.current = target_weights.copy()

    def needs_rebalance(self) -> bool:
        return (self.current - self.target).abs().max() > self.threshold

    def new_weights(self, new_returns: pd.Series) -> pd.Series:
        self.current = self.current * (1 + new_returns)
        self.current = self.current / self.current.sum()
        if self.needs_rebalance():
            self.current = self.target
        return self.current
```
