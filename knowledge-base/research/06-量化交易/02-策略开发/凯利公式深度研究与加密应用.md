## 0. 摘要

凯利公式（Kelly criterion）是**最大化对数财富长期增长率**的仓位规则，由 John Larry Kelly Jr. 于 1956 年在 Bell Labs 发表的一篇 10 页通信中给出——其原始论文是**信息论**论文而非金融论文，揭示了"最大化金钱增长率 = 最大化信道信息传输率"这一深刻等价。

本文给出：

1. **理论谱系**：从 Kelly 1956 → Breiman 1961 → Latané 1959 → Hakansson-Ziemba → MacLean-Thorp-Ziemba → Busseti-Ryu-Boyd 2016 的完整学术链；
2. **多形式公式**：二项式/多结果/连续分布/多资产/带交易成本/带杠杆约束/带回撤约束的 8 种 Kelly 变体；
3. **加密应用**：现货、永续合约 funding rate、波动率目标、Memecoin 极端右偏分布下的 Kelly 校准；
4. **实战陷阱**：参数估计误差、肥尾破产、流动性约束、回撤、漂移、Black Swan；以及**Half-Kelly / Quarter-Kelly** 的实用折衷；
5. **Python 实现**：9 段可直接运行的代码（经典 Kelly → 多资产 → 回撤约束 → 在线 Bayesian 估计 → 加密 funding-rate Kelly → 与 Backtrader 集成）；

### 1.2 关键洞察：信息率 = 财富增长率

Kelly 解决的是 Shannon 留下的一个语义问题：信道容量 $C$ 的"bit/s"对发送者有什么用？

答案：如果你把赌注下在一个"消息-赌局"上，使**下注的资本**与**信道的输出分布**匹配，那么**财富的对数增长率**就恰好等于信道的**信息传输率**。即：

$$
g(f^*) = C - H(\text{source})
$$

- 最大化金钱 = 最大化信息；
- 这把 Shannon 信息论与长期投资增长理论**严密统一**。

### 1.3 三条核心公式

**(1) 对数增长率：**

$$
g(f) = \mathbb{E}\!\left[\log\frac{W_{n+1}}{W_n}\right] = \sum_i p_i \log(b_i f + 1)
$$

**(2) 一阶最优条件（Kelly's proportional betting condition）：**

$$
\sum_i \frac{p_i b_i}{1 + b_i f^*} = 0
$$

**(3) 二项式等额赌局最简形式（最常被引用）：**

$$
f^* = p - \frac{q}{b} = p - \frac{1-p}{b}
$$

其中 $p$ 是胜率，$q = 1-p$，$b$ 是净赔率（盈利/亏损）。等额赌（$b=1$）时退化为 $f^* = 2p - 1$。

### 1.4 Breiman 大数定律：渐近最优性

Leo Breiman 1961 年的附记（"Optimal Gambling Systems for Favorable Games"）证明：

- **对数增长率**：$g(f^*) \geq g(f), \forall f$；
- **几何中位数**：Kelly 策略的财富中位数在 $n \to \infty$ 时**严格大于**任何其他策略；
- **首达时（首次翻倍/破产）**：Kelly 也有最优的渐近分布。

这一结果是 Kelly 公式的"理论护城河"——**它是唯一在长期下严格最优的仓位规则**。

## 3. Kelly 公式的 8 种变体

### 3.1 二项式（最经典）

$$
f^* = p - \frac{q}{b} = p - \frac{1-p}{b}
$$

| 胜率 $p$ | 净赔率 $b$ | Kelly $f^*$ |
|---|---|---|
| 0.60 | 1.0 | 0.20（20%） |
| 0.55 | 1.0 | 0.10 |
| 0.51 | 1.0 | 0.02 |
| 0.50 | 1.0 | 0（无 edge） |
| 0.45 | 1.0 | -0.10（反向） |
| 0.60 | 2.0 | 0.40 |
| 0.60 | 0.5 | 0.20（仍然 0.20） |

> 注意：只要 $pb > q$，Kelly 都是正数，**与 $b$ 大小无直接关系**（公式看 $p-q/b$）。

### 3.2 多结果（多项式）

$$
f_i^* = \arg\max_f \sum_j p_j \log(1 + b_{ij} f_i)
$$

每个赌局独立求最优；多资产时为向量形式。

### 3.3 连续分布

若 $X$ 是单期收益（带密度 $f_X$），则最优解满足：

$$
\mathbb{E}\!\left[\frac{X}{1 + f^* X}\right] = 0
$$

求解需迭代（Newton 法 / 二分），无闭式解。

### 3.4 多资产（向量形式）

$$
\mathbf{f}^* = \arg\max_{\mathbf{f}} \mathbb{E}\!\left[\log(\mathbf{1} + \mathbf{R}^\top \mathbf{f})\right]
$$

其中 $\mathbf{R} \in \mathbb{R}^T$ 是收益向量。一阶条件：

$$
\mathbb{E}\!\left[\frac{\mathbf{R}}{\mathbf{1} + \mathbf{R}^\top \mathbf{f}^*}\right] = \mathbf{0}
$$

实践中用 CVXPY / scipy.optimize 求解凸优化。

### 3.5 带交易成本（Davis-Lleo / Brockwell）

Davis & Lleo 2011 给出在已有持仓 $h_0$ 下的调整：

$$
f^* = h_0 + \frac{1}{2c} \left( \text{edge} - c \cdot |h_0| \right)
$$

其中 $c$ 是单边交易成本率。这告诉 Kelly 交易者"成本越高，f 调整越小"。

### 3.6 带杠杆约束

若 $|f| \le L$，则解在约束边界上：

$$
f^*_{\text{clipped}} = \text{clip}(f^*_{\text{unconstrained}},\; -L, L)
$$

### 3.7 带回撤约束（Busseti-Ryu-Boyd 2016）

Busseti 等的 **risk-constrained Kelly** 严格优于分数 Kelly：

$$
\begin{aligned}
\max_f \quad & \mathbb{E}[\log(W_T)] \\
\text{s.t.} \quad & \Pr(W_t \le \alpha W_0) \le \delta \quad \forall t
\end{aligned}
$$

凸重构后，**单参数** $\delta$ 控制最大回撤概率。**实证上：相同回撤水平下，risk-constrained Kelly 的增长率比 half-Kelly 高 1.5-2×**。

### 3.8 鲁棒 / 分布鲁棒 Kelly（Sun-Boyd 2018）

假设真实分布在最坏 Wasserstein 球内：

$$
f^* = \arg\max_f \min_{P \in \mathcal{B}_\epsilon(\hat{P})} \mathbb{E}_P[\log(1 + R f)]
$$

防止对样本分布的过拟合。

## 4. 加密应用：从现货到永续合约 funding

### 4.1 现货仓位：单币种

把"等额赌局"换成 BTC 现货的多空策略：

- 胜率 $p = $ 信号命中率（如 RSI 30 以下 + 趋势确认 → 5 日内 +5%）；
- 赔率 $b = $ 平均盈利 / 平均亏损；
- Kelly $f^* = p - q/b$。

**示例**：Backtest 显示信号命中 0.55，平均盈利 +6%，平均亏损 -4%，$b = 1.5$，则 $f^* = 0.55 - 0.45/1.5 = 0.25$。Half-Kelly = 12.5% 单仓。

### 4.2 多币种组合

按 [Kelly, Malamud, Pedersen 2024](https://arxiv.org/abs/2402.05256) 的 **principal portfolios** 框架，先做主成分降维，再在主成分空间内做 Kelly 优化——可大幅降维、避免协方差估计误差。

### 4.3 永续合约 funding rate Kelly

永续合约的 8 小时 funding rate（$r_f$）是**持仓成本**。有效 Kelly 公式：

$$
f^*_{\text{perp}} = f^*_{\text{spot}} \cdot \frac{1}{1 + r_f \cdot T_{\text{hold}}}
$$

**示例**：年化 funding 8%，预期持仓 30 天，$f^*_{\text{perp}} = 0.25 \times 0.994 = 0.2485$——**影响很小**（<1%）。

**但若 funding 突变为负**（即 short 付 long 报酬），做空策略的 $f^*$ **反向放大**——这是 funding-rate arbitrage 策略的 Kelly 核心。

### 4.4 波动率目标 + Kelly

对杠杆/保证金账户，目标波动率 $\sigma_{\text{target}}$ 与 Kelly 的关系：

$$
f^* = \frac{\mu - r}{\sigma^2} \cdot \frac{\sigma_{\text{target}}}{\sigma}
$$

- 第一项是连续 Kelly（$f^* = \mu/\sigma^2$ 当 $r=0$）；
- 第二项是 vol targeting 缩放。

**示例**：策略 Sharpe = 0.5，月波动率 8%，目标月波动率 3%，则 $f^* = 0.5 \times 3/8 = 0.1875$。

### 4.5 Memecoin 极端右偏

Memecoin 的"2 倍中位数 + 100 倍长尾"分布**违反 Kelly 的所有标准假设**（高斯、平稳、独立）。在这种情况下：

1. **截断 Kelly**（truncated Kelly）：在 $f \le f_{\max}$ 区间求解；
2. **CVaR-Kelly**：用条件 VaR 而非方差做约束；
3. **保守 Kelly**：取 $f^*_{\text{conservative}} = f^*_{\text{naive}} / 5$（quarter-Kelly 都不够）。

经验法则：**对 Memecoin，永远不要超过 bankroll 的 1%**。

## 5. 实战陷阱：为什么 90% 的人用错 Kelly

### 5.1 七大陷阱清单

| 陷阱 | 表现 | 解决 |
|---|---|---|
| **1. 参数估计误差** | 样本胜率 0.55，真实 0.52 → Kelly 从 0.10 跌至 0.04 | 在线 Bayesian 估计 + 缩小 |
| **2. 肥尾（Black Swan）** | 学生 $t$ 分布 $\nu=3$ 一次 10σ 事件破产 Kelly | 显式用 $t$ / power-law 分布 |
| **3. 流动性约束** | Kelly 说 f=1.5，但最大可买 f=0.3 | 约束优化 + 流动性加权 |
| **4. 回撤（drawdown）** | Full-Kelly 历史回测 DD>50% | Half / Quarter / risk-constrained |
| **5. 漂移（time-varying edge）** | 胜率随时间漂移，固定 Kelly 失效 | 滚动估计 + 隐变量模型 |
| **6. 非平稳（regime switch）** | 牛市 Kelly 跟熊市 Kelly 完全不同 | HMM 状态检测 + 状态内 Kelly |
| **7. 心理压力** | 模拟 OK，真实交易扛不住 50% DD | **永远用 half-Kelly 起步** |

### 5.2 关键经验数据（来自 MacLean-Thorp-Ziemba 2010）

| 策略 | S&P 500 1962-2007 年化 log-growth | 最大回撤 |
|---|---|---|
| **Full Kelly** | ~12% | **~50%** |
| **Half Kelly** | ~9% | ~25% |
| **Quarter Kelly** | ~6% | ~12% |

**结论**：Half-Kelly 损失 ~25% 增长率，换回 ~50% 回撤削减——**绝大多数实际交易者应停在这里**。

### 5.3 Thorp 的 Princeton Newport Partners 案例

Ed Thorp 的对冲基金 1969-1988 年化 ~20%（累积 ~2,800%），无亏损年。**实际跑的是 half-Kelly 或更低**——Poundstone 在 *Fortune's Formula*（2005）中明确指出 Thorp "从不接近全 Kelly"。

**这反驳了一个常见神话：Kelly = 最优，但 Kelly 在真实交易中**不可行**。实用解 = 分数 Kelly。**

## 6. Python 实现（9 段代码）

### 6.1 基础 Kelly（二项式）

```python
def kelly_binary(p: float, b: float) -> float:
    """二项式 Kelly: 胜率 p, 净赔率 b (盈利/亏损)."""
    q = 1 - p
    return p - q / b

# 例: 60% 胜率, 1:1 赔率
print(kelly_binary(0.60, 1.0))  # 0.20
print(kelly_binary(0.55, 1.0))  # 0.10
print(kelly_binary(0.51, 1.0))  # 0.02
```

### 6.2 多结果 Kelly

```python
import numpy as np
from scipy.optimize import minimize_scalar

def kelly_multi(p: np.ndarray, b: np.ndarray) -> float:
    """多结果 Kelly: 概率 p[i], 净赔率 b[i] (单变量决策)."""
    def neg_growth(f):
        # f* = 1 + sum p[i] log(1 + b[i] f)
        if np.any(1 + b * f <= 0):
            return np.inf
        return -np.sum(p * np.log(1 + b * f))
    res = minimize_scalar(neg_growth, bounds=(-0.99, 10.0), method='bounded')
    return res.x
```

### 6.3 连续分布 Kelly（数值求解）

```python
def kelly_continuous(mu: float, sigma: float) -> float:
    """正态近似: f* ≈ mu / sigma^2."""
    return mu / sigma**2

# 期望收益 mu=0.10, 波动率 sigma=0.20
print(kelly_continuous(0.10, 0.20))  # 2.5
# 注: 这要求 mu 单位与 sigma 平方单位一致 (如年化)
```

### 6.4 多资产 Kelly（CVXPY 凸优化）

```python
import cvxpy as cp
import numpy as np

def kelly_multi_asset(R: np.ndarray) -> np.ndarray:
    """R: T×N 收益矩阵 (T 期, N 资产). 求 f ∈ R^N."""
    T, N = R.shape
    f = cp.Variable(N)
    # 最大化 E[log(1 + R @ f)]
    wealth = 1 + R @ f
    # 用样本均值近似
    objective = cp.Maximize(cp.sum(cp.log(wealth)) / T)
    constraints = [wealth >= 1e-3, f >= -2, f <= 2]  # 防破产, 限杠杆
    prob = cp.Problem(objective, constraints)
    prob.solve(solver=cp.SCS)
    return f.value
```

### 6.5 在线 Bayesian Kelly 估计

```python
class BayesianKelly:
    """在线估计胜率 p (Beta 后验) + Kelly 仓位."""
    def __init__(self, p0: float = 0.5, prior_strength: float = 10):
        self.alpha = p0 * prior_strength
        self.beta = (1 - p0) * prior_strength
    
    def update(self, win: bool):
        if win:
            self.alpha += 1
        else:
            self.beta += 1
    
    @property
    def p_mean(self) -> float:
        return self.alpha / (self.alpha + self.beta)
    
    def kelly(self, b: float = 1.0, fraction: float = 0.5) -> float:
        """fraction: half-Kelly / full-Kelly 缩放."""
        p = self.p_mean
        return fraction * (p - (1 - p) / b)

# 用法
bk = BayesianKelly()
for outcome in [True, True, False, True, True, False, True]:
    bk.update(outcome)
print(f"在线胜率估计: {bk.p_mean:.3f}, Half-Kelly: {bk.kelly():.3f}")
```

### 6.6 永续合约 Funding-rate Kelly

```python
def kelly_perp(f_spot: float, funding_annual: float, hold_days: int) -> float:
    """永续合约 Kelly 调整.
    funding_annual: 年化 funding 率 (持仓方收入为正)
    hold_days: 预期持仓天数
    """
    funding_cost = funding_annual * hold_days / 365
    return f_spot / (1 + funding_cost)

# 例: 现货 Kelly=0.25, 年化 funding=8%, 持仓 30 天
print(kelly_perp(0.25, 0.08, 30))  # 0.2485
```

### 6.7 波动率目标 Kelly

```python
def kelly_vol_target(mu: float, sigma: float, sigma_target: float) -> float:
    """波动率目标 Kelly: f = (mu/sigma^2) * (sigma_target/sigma)."""
    kelly_full = mu / sigma**2
    return kelly_full * (sigma_target / sigma)

# 策略 Sharpe=0.5, 月 sigma=8%, 目标月 sigma=3%
mu, sigma, st = 0.04, 0.08, 0.03
print(kelly_vol_target(mu, sigma, st))  # 0.1875
```

### 6.8 回撤约束 Kelly（Busseti-Ryu-Boyd 简化版）

```python
def kelly_with_dd_cap(p: float, b: float, max_dd: float = 0.30) -> float:
    """回撤上限 Kelly: 通过二项式 + 约束缩放近似.
    简化公式: f_eff = f_full * sqrt(max_dd / 0.50)
    基于: full-Kelly 历史回撤约 50% (S&P 500 经验值).
    """
    f_full = p - (1 - p) / b
    scale = (max_dd / 0.50) ** 0.5
    return f_full * min(scale, 1.0)

# 胜率 55%, 赔率 1:1, 最大允许回撤 25%
print(kelly_with_dd_cap(0.55, 1.0, 0.25))  # 0.0707
```

### 6.9 Backtrader 集成（实战级）

```python
import backtrader as bt

class KellyPositionSizer(bt.Sizer):
    """Backtrader 的 Kelly 仓位管理器.
    需配合 Strategy 在 next() 中更新胜率估计.
    """
    params = (
        ('lookback', 50),       # 估计窗口
        ('fraction', 0.5),      # half-Kelly
        ('b', 1.0),             # 赔率
        ('max_pos', 0.5),       # 仓位硬上限
    )
    
    def __init__(self):
        self.wins = 0
        self.losses = 0
        self.trades = []
    
    def notify_trade(self, trade):
        if trade.isclosed:
            self.trades.append(trade.pnl)
            if trade.pnl > 0:
                self.wins += 1
            else:
                self.losses += 1
    
    def _getsizing(self, data, isbuy):
        n = self.wins + self.losses
        if n < 20:  # 冷启动期
            return 0
        p = self.wins / n
        kelly = self.p.fraction * (p - (1 - p) / self.p.b)
        kelly = max(0, min(kelly, self.p.max_pos))
        size = self.broker.getvalue() * kelly / data.close[0]
        return int(size)
```

## 9. 实用检查清单（部署前必读）

部署 Kelly 策略到生产前，问自己以下 12 个问题（**任一答 "no" → 退回 half-Kelly**）：

1. 胜率/赔率的**样本量** ≥ 100 笔交易？
2. 样本**自相关性**检验通过（Ljung-Box p > 0.05）？
3. **回测**最大回撤 ≤ 30%？
4. **滑点/手续费**后 Kelly 仍为正？
5. 策略**半衰期** ≤ 历史数据长度 1/4？
6. 流动性**充足**到允许 Kelly 仓位建仓/平仓不冲击价格？
7. **资金费率** / 借贷成本可承担？
8. 心理上**能承受**模拟显示的最大回撤（real money 痛 +50%）？
9. 策略在**至少 2 个 regime**（牛/熊）下 Kelly 仍为正？
10. 有**自动熔断**：连续 N 笔亏损自动切到 quarter-Kelly？
11. 监控**胜率漂移**：胜率变化 > 10% 自动报警？
12. 仓位**硬上限** ≤ 20% bankroll？

✅ 12/12 = 勉强可上 full-Kelly；
✅ 9-11 = 限 half-Kelly；
✅ 6-8 = 限 quarter-Kelly；
❌ <6 = 不要上 Kelly 框架，改用固定 1% 仓位。
