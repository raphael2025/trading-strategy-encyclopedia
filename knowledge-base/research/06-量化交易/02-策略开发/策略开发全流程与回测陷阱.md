### 2. 各阶段产出

|------|------|--------|

| 数据准备 | 可用数据集 | 10% |
| 信号构建 | 信号代码 | 30% |
| 组合构建 | 仓位规则 | 30% |
| 回测验证 | 回测报告 | 50% |

**结论**：从假设到稳定运行，**80%+ 策略失败**，这是正常现象。

## 二、假设形成

### 1. 假设来源

**市场观察**：
- "BTC 减半后 12 个月通常上涨"
- "Funding Rate > 0.05% 时容易发生 long squeeze"

**学术文献**：
- Jegadeesh & Titman (1993) 动量效应
- Carhart (1997) 四因子模型
- Frazzini (2006) 反向处置效应

**行业研究**：
- Glassnode 链上报告
- Binance Research
- a16z Crypto 报告

### 2. 假设质量评估

**好假设的特征**：
- 有**经济学逻辑**支撑（不只是统计相关）
- **可证伪**（如果发生 X，则 Y 必须成立）
- **有限范围**（不是"任何时候都有效"）
- **不同环境**鲁棒（牛熊都有效）

**坏假设的特征**：
- 只有数据挖掘支持
- "事后诸葛亮"
- 难以定义
- 只在特定时段有效

**示例**：

✅ **好假设**：
> "当 BTC MVRV > 3.5 时，未来 6 个月平均回撤 -30%+，因此应该减仓"
> - 经济学逻辑：MVRV 衡量市场价值 vs 已实现价值，高位 = 投机过热
> - 可证伪：可以测试历史数据
> - 有限范围：明确"MVRV > 3.5"

❌ **坏假设**：
> "BTC 涨到 100,000 美元"
> - 没有经济逻辑
> - 不可证伪
> - 不是策略，是预测

## 三、数据准备

### 1. 数据分类

| 类型 | 来源 | 频率 |
|------|------|------|
| 行情数据 | CEX API | tick / OHLCV |
| 链上数据 | Glassnode / CryptoQuant | 每日 |
| 衍生品数据 | Coinglass | tick |
| 社交数据 | Twitter API | 实时 |
| 宏观经济 | FRED / Bloomberg | 每日/周 |
| 公司/项目数据 | 公告 / GitHub | 实时 |

### 2. 数据清洗

**常见问题**：
- **缺失值**：交易所 API 故障
- **异常值**：极端行情、合约标记错误
- **重复值**：分页错误
- **时区问题**：UTC vs 本地时间
- **对齐问题**：不同数据源时间戳不同

**清洗方法**：
```python
# 示例：K 线数据清洗
df = df.drop_duplicates()
df = df[df['volume'] > 0]  # 去除异常
df = df.fillna(method='ffill')  # 前向填充
df = df.tz_localize('UTC')  # 统一时区
```

### 3. 数据存储

**最佳实践**：
- **原始数据**：保留原始 CSV/Parquet
- **清洗后数据**：保存为统一格式
- **元数据**：记录数据来源、时间戳、版本

**数据库选择**：
- 大量 tick 数据：ClickHouse, QuestDB
- K 线数据：TimescaleDB, InfluxDB
- 链上数据：PostgreSQL + TimescaleDB 扩展

### 4. 数据陷阱

**回看偏差（Lookback Bias）**：
- 用了未来数据
- 例：用今日 4pm 的价格做今日 3pm 的决策
- **防御**：严格时间戳对齐

**生存者偏差**：
- 只取"现存"的币
- 忽略已下架/归零的
- **防御**：用历史全量数据

**前视偏差（Look-ahead Bias）**：
- 用今日发布的指标做昨日的决策
- 例：用今日 8am 公布的就业数据做昨日的交易
- **防御**：明确信息发布时刻

## 四、信号构建

### 1. 信号类型

**技术信号**：
- 均线交叉（MA crossover）
- RSI 超买超卖
- 布林带（Bollinger Bands）
- MACD

**基本面信号**：
- 估值（MVRV, NVT）
- 网络增长（活跃地址、交易量）
- 链上活跃度

**资金流信号**：
- 交易所净流入/流出
- 稳定币供应
- 巨鲸动向
- Funding Rate

**情绪信号**：
- 社交媒体（Twitter, Reddit）
- 新闻情绪
- 谷歌趋势
- 恐惧贪婪指数

### 2. 信号评估指标

**IC（Information Coefficient）**：
- 信号与未来收益的相关系数
- IC > 0.05：可用
- IC > 0.10：优秀

**Rank IC**：
- 信号秩与未来收益秩的相关系数
- 比 IC 更稳健
- Rank IC > 0.10：优秀

**IC 衰减**：
- 信号的预测能力随时间衰减
- 评估"半衰期"

### 3. 因子合成

**简单加权**：
- 各因子等权
- 各因子按 IC 加权

**机器学习方法**：
- 线性回归
- 随机森林
- 梯度提升（XGBoost）
- 神经网络

**警告**：
- 避免过拟合
- 样本外测试
- Walk-Forward 验证

### 4. 因子衰减监控

**监控指标**：
- 当前 IC vs 历史 IC
- 因子值分布变化
- 实际表现 vs 预期

**退役时机**：
- IC 持续 6 个月低于 0.02
- 因子失效有理论解释

## 五、组合构建

### 1. 基本方法

**等权（1/N）**：
- 每个资产 1/N
- 简单但不最优

**市值加权**：
- 按市值分配
- 倾向大盘币

**波动率倒数加权**：
- 仓位 ∝ 1/σ
- 控制波动率贡献

**风险平价（Risk Parity）**：
- 每个资产贡献等量风险
- Bridgewater 著名

**最大夏普**：
- Markowitz 优化
- 估计输入参数（μ, σ, ρ）
- 容易过拟合

**Black-Litterman**：
- 融合主观观点
- 适合大资金

### 2. 凯利公式（Kelly Criterion）

**公式**：
- f* = (p × b - q) / b
- p = 胜率
- q = 1 - p
- b = 盈亏比

**例**：
- 胜率 60%（p=0.6）
- 盈亏比 2:1（b=2）
- f* = (0.6×2 - 0.4)/2 = 0.4 = 40%
- **半凯利**：20%

**警告**：
- 实际参数不准确
- **实战**：用 1/4 Kelly 起步

### 3. 仓位约束

**单资产上限**：5-10%
**单策略上限**：20%
**总杠杆上限**：2-3x
**行业集中度**：30%
**流动性约束**：每日交易量 < 1%

## 六、回测验证

### 1. 回测类型

**单次回测（In-sample）**：
- 整个数据集
- 快速但容易过拟合

**样本外测试（Out-of-sample）**：
- 训练集 + 测试集
- 数据集 70/30 切分

**Walk-Forward Analysis**：
- 滚动窗口
- 多轮样本外
- 最严格

**Cross-Validation**：
- 时间序列 K-fold
- 保留时间顺序

### 2. Walk-Forward 示例

```
┌────────────────────────────────┐
│  Train      │ Test              │
│  2017-2019   │ 2020             │
└────────────────────────────────┘
        ↓
┌────────────────────────────────┐
│  Train            │ Test        │
│  2017-2020        │ 2021        │
└────────────────────────────────┘
        ↓
┌────────────────────────────────┐
│  Train                │ Test    │
│  2017-2021            │ 2022    │
└────────────────────────────────┘
```

### 3. 性能评估指标

**收益指标**：
- **总收益**：累计 P&L
- **CAGR**：年化复合增长率
- **年化收益**：（1+总收益）^(1/年数) - 1

**风险指标**：
- **Sharpe Ratio**：超额收益/波动率
  - > 1.5：合格
  - > 2.0：优秀
- **Sortino Ratio**：下行波动调整
- **Calmar Ratio**：年化收益/最大回撤
- **Max Drawdown**：最大回撤
- **Volatility**：年化波动率

**交易指标**：
- **Win Rate**：胜率
- **Profit Factor**：总盈利/总亏损
- **Avg Win/Loss Ratio**：平均盈亏比
- **Number of Trades**：交易次数（< 30 不可信）
- **Avg Holding Period**：平均持仓期

### 4. 关键基准对比

| 基准 | 加密对应 | 超越难度 |
|------|----------|----------|
| 标普 500 | BTC HODL | 难 |
| 60/40 组合 | 50% BTC + 50% 稳定币 | 中 |
| 现金 | 0% 仓位 | 容易 |
| 买入持有 | 长期持有 | 中 |

## 七、常见回测陷阱

### 1. 未来数据（Look-ahead Bias）

**症状**：回测 Sharpe 异常高（> 3），实盘崩

**典型案例**：
- 用今日收盘价做今日交易
- 用今日 8am 公布的指标做今日 1am 的决策
- 用未来的指数成分股做回测

**防御**：
- 严格时间戳对齐
- 数据版本控制
- 模拟实时数据流

### 2. 过拟合（Overfitting）

**症状**：参数越多越好；样本外崩溃

**典型案例**：
- 100 个参数，只用 1 年数据
- 优化到 0.0001 粒度的阈值

**防御**：
- 少参数（< 10）
- Walk-Forward
- 样本量 > 参数数 × 100
- Out-of-sample 测试

### 3. 生存者偏差

**症状**：回测好，实盘差

**典型案例**：
- 只取"现存"的币（如 BTC, ETH）
- 忽略归零的 ICO 代币

**防御**：
- 用历史全量数据
- 包含下架/归零代币
- 链上数据时考虑全部历史

### 4. 交易摩擦忽略

**症状**：回测时未扣手续费、滑点、资金费率

**典型案例**：
- 0.1% 手续费忽略
- BTC 永续 funding 0.05%/8h 忽略
- 大单冲击成本忽略

**防御**：
- 强制 0.05-0.10% 单边成本
- 流动性调整（市价单 vs 限价单）
- 滑点模型（基于订单簿深度）

### 5. 数据窥探（Data Snooping）

**症状**：用同一数据集测试 1000 个策略，选最好的

**典型案例**：
- 测试 100 个技术指标，挑 Sharpe 最高的
- 多个参数组合，事后挑最好的

**防御**：
- Bonferroni 校正
- White Reality Check
- 严格 Walk-Forward

### 6. 红利再投资假设错误

**症状**：用高分红率币种，过度乐观

**典型案例**：
- 假设 staking 收益自动复利
- 假设空投 100% 抓到

**防御**：
- 保守估计
- 实际复投频率

### 7. 头寸规模错误

**症状**：固定 100% 仓位；忽略保证金

**典型案例**：
- 永远满仓
- 不考虑资金费率

**防御**：
- 资金管理规则
- 保证金监控
- 风险预算

## 八、性能归因

### 1. Sharpe 分解

**总 Sharpe = α (超额) + β × 市场**

例：
- BTC 收益 30%
- 策略收益 60%
- 波动率 30%
- 假设 BTC 收益 30%，策略 Beta 1.5
- α = 60% - 1.5 × 30% = 15%
- **纯 Alpha 15%**（实际是 +30% 中的额外部分）

### 2. Brinson 归因

**配置效应（Allocation）**：选择哪个资产
**选择效应（Selection）**：何时交易
**交互效应**：两者结合

### 3. 时间维度归因

- 不同时段（月/季度）表现
- 不同市场状态（牛/熊/横盘）表现
- 不同事件（FOMC, ETF）表现

### 4. 风险归因

- 行业风险贡献
- 因子风险贡献
- 特质风险贡献

## 九、加密量化策略实例

### 1. 动量策略（Momentum）

**假设**：过去赢家继续赢

**代码骨架**（伪代码）：
```python
# 1. 计算 20 天收益率
returns = df['close'].pct_change(20)

# 2. 排序
top = returns.nlargest(5).index  # Top 5

# 3. 等权持有
weights = {asset: 0.2 for asset in top}

# 4. 周/月再平衡
```

**历史表现**（加密）：
- 年化 30-50%
- Sharpe 1.0-1.8
- Max DD 30-40%

**陷阱**：仅在趋势市场有效；震荡市失效

### 2. 均值回归（Mean Reversion）

**假设**：极端价格回归均值

**代码骨架**：
```python
# 1. 计算 RSI
rsi = compute_rsi(df, 14)

# 2. 信号：RSI < 30 买入
signal = (rsi < 30).astype(int)

# 3. 持有 5 天
# 4. 止盈止损
```

**历史表现**：
- 年化 20-40%
- Sharpe 1.2-2.0
- 在横盘市场更好

**陷阱**：在趋势市场会被反复止损

### 3. 链上+价格多因子

**假设**：综合价格动量 + 链上基本面

```python
# 因子 1：MVRV
factor1 = -df['MVRV']  # MVRV 越低，越加仓

# 因子 2：交易所净流出
factor2 = df['exchange_netflow']

# 因子 3：Funding Rate
factor3 = -df['funding_rate']  # funding 越高，越减仓

# 综合得分
score = factor1 * 0.4 + factor2 * 0.4 + factor3 * 0.2
```

**历史表现**：
- 年化 40-60%
- Sharpe 1.5-2.5

**陷阱**：链上数据噪声大；不同币种需校准

### 4. 资金费率套利

**假设**：Funding 极端化反向回归

**逻辑**：
- Funding > 0.05%：做空永续 + 买入现货
- Funding < -0.05%：做多永续 + 借币卖出

**收益**：
- 典型年化 10-30%
- **风险**：基差扩大 → 爆仓

### 5. 期现套利

**假设**：现货 - 季度期货基差回归

**逻辑**：
- 买入现货 + 卖出季度期货
- 持有到期交割
- 赚基差收敛

**收益**：
- 年化 5-15%
- **风险**：交割失败、流动性

## 十、上线与监控

### 1. 上线阶段

**Step 1：模拟盘（2-4 周）**
- 完全模拟，不接真实资金
- 验证技术栈

**Step 2：小资金实盘（1-3 个月）**
- 10-20% 目标资金
- 监控实际 vs 回测

**Step 3：逐步加仓**
- 表现稳定后加到 50-100%

### 2. 监控指标

**业务**：
- P&L（日/周/月/累计）
- Sharpe（滚动）
- Max DD（实时）
- 仓位（当前）

**技术**：
- API 延迟
- 订单执行时间
- 数据延迟
- 系统资源

**市场**：
- IV（期权）
- Funding Rate
- 交易所余额
- 链上异常

### 3. 异常处理

**P&L 异常**：
- 触发 5% 日亏 → 自动报警
- 触发 10% 累计亏 → 减小仓位
- 触发 20% 累计亏 → 停止策略

**系统异常**：
- API 失败 → 降级到备用
- 数据延迟 → 暂停交易
- Kill Switch → 全部平仓

## A.1 三重障碍标签 (Triple Barrier Method)

```python
"""
传统固定窗口标签：未来 N 根 K 线收益率 → 噪声大
三重障碍：止盈/止损/到期 — 更贴近交易决策
- 上方水平线 = 止盈 (take-profit)
- 下方水平线 = 止损 (stop-loss)
- 垂直线 = 最大持有期 (max holding)
- 谁先触碰 → 对应标签
"""
import numpy as np
import pandas as pd


def triple_barrier_labels(prices: pd.Series, events: pd.DataFrame,
                          upper: float, lower: float, max_holding: int) -> pd.DataFrame:
    """
    events: 至少含 't1' 列 (label 起始时间)
    returns: DataFrame 含 't1' (label 结束时间) 和 'bin' (-1/0/1)
    """
    labels = []
    for t0, row in events.iterrows():
        path = prices.loc[t0:row['t1']].iloc[1: max_holding + 1]
        ret = (path / prices.loc[t0]) - 1
        up = ret[ret >= upper].index.min() if (ret >= upper).any() else pd.NaT
        dn = ret[ret <= -lower].index.min() if (ret <= -lower).any() else pd.NaT
        exp = path.index[-1]
        first = min([x for x in [up, dn, exp] if pd.notna(x)])
        bin_ = 1 if first == up else (-1 if first == dn else 0)
        labels.append({'t0': t0, 't1': first, 'bin': bin_, 'ret': ret[first] - 0})
    return pd.DataFrame(labels).set_index('t0')
```

## A.2 Meta-Labeling（元学习二阶段模型）

```python
"""
主模型预测方向（3 类），次模型预测置信度（2 类）
- 主模型: 长/短/不操作
- 次模型 (side): 该信号该不该下手 (binary)
- 好处：次模型可以额外叠加（信号 + 基本面 + 微观结构）
"""
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import KFold
import numpy as np
import pandas as pd


def meta_labeling(X: pd.DataFrame, y_side: pd.Series,
                   y_size: pd.Series = None) -> dict:
    side_model = RandomForestClassifier(n_estimators=500, max_depth=4,
                                         min_samples_leaf=50, n_jobs=-1)
    cv = KFold(n_splits=5, shuffle=False)
    oof_proba = pd.Series(np.nan, index=X.index)
    for train_idx, test_idx in cv.split(X):
        side_model.fit(X.iloc[train_idx], y_side.iloc[train_idx])
        oof_proba.iloc[test_idx] = side_model.predict_proba(X.iloc[test_idx])[:, 1]
    side_model.fit(X, y_side)
    return {"side_model": side_model, "oof_proba": oof_proba}
```

## A.3 Combinatorial Purged CV (CPCV)

```python
"""
K-Fold 泄露问题：测试集已"看过"训练信息
CPCV (Lopez de Prado 2018)：
1) 数据先 purge (清除重叠样本)
2) embargo (延后评估) 防止泄露
3) 训练/测试组所有组合穷举
- 估计策略在不同路径下的真实过拟合概率 (PBO)
"""
import numpy as np
import pandas as pd
from sklearn.model_selection import KFold
from itertools import combinations


def cpcv_paths(n_splits: int, n_test_splits: int = 2) -> list:
    """返回所有 (train_idx, test_idx) 路径"""
    kf = KFold(n_splits=n_splits, shuffle=False)
    splits = list(kf.split(np.arange(n_splits * 10)))
    paths = []
    for combo in combinations(range(n_splits), n_test_splits):
        test = np.concatenate([splits[i][1] for i in combo])
        train = np.concatenate([splits[i][0] for i in range(n_splits) if i not in combo])
        paths.append((train, test))
    return paths
```

## A.4 Deflated Sharpe Ratio (DSR)

```python
"""
普通 Sharpe Ratio 没考虑：
1) 非正态分布 (fat tails)
2) 多重检验 (尝试 N 个策略，挑 SR 最高的)
Bailey & López de Prado (2014) 提出 DSR
DSR = [SR̂ - SR*] * sqrt(T-1) / [1 - γ₃ SR̂ + (γ₄-1)/4 SR̂²]
- SR*: 多次实验后经多重检验校正的阈值
- γ₃, γ₄: 偏度/峰度
"""
import numpy as np
import pandas as pd
from scipy.stats import skew, kurtosis, norm


def deflated_sharpe_ratio(returns: pd.Series, sr_benchmark: float = 0,
                          n_trials: int = 100) -> dict:
    sr_hat = returns.mean() / returns.std() * np.sqrt(252)
    skew_ = skew(returns.dropna())
    kurt_ = kurtosis(returns.dropna(), fisher=False)
    n = len(returns)
    sr_max = (1 - np.euler_gamma) * norm.ppf(1 - 1/n_trials) + \
             np.euler_gamma * norm.ppf(1 - 1/(n_trials * np.e))
    dsr = (sr_hat - sr_benchmark) * np.sqrt(n - 1) / \
          np.sqrt(1 - skew_ * sr_hat + (kurt_ - 1) / 4 * sr_hat ** 2)
    p_value = norm.cdf(-dsr)
    return {"sharpe": sr_hat, "dsr": dsr, "p_value": p_value,
            "sr_threshold": sr_max}
```

## A.5 PBO（Probability of Backtest Overfitting）

```python
"""
CPCV 训练所有组合，记录每路径最优 SR
PBO = 比例(策略在 IS 表现好 → OOS 表现差)
PBO > 0.5 → 高概率过拟合
"""
import numpy as np
import pandas as pd


def probability_of_overfitting(returns_matrix: pd.DataFrame,
                                n_paths: int = 16) -> float:
    """
    returns_matrix: 行=时间, 列=策略
    """
    paths = cpcv_paths(n_splits=int(np.sqrt(n_paths)))
    logit_loss_oos = []
    for train_idx, test_idx in paths[:n_paths]:
        is_best = returns_matrix.iloc[train_idx].mean().idxmax()
        oos_logit = np.log((1 + returns_matrix.iloc[test_idx][is_best]) /
                            (1 - returns_matrix.iloc[test_idx][is_best])).sum()
        logit_loss_oos.append(oos_logit)
    return float(np.mean(np.array(logit_loss_oos) < 0))
```

## A.6 多重检验校正 (Holm-Bonferroni)

```python
"""
N 个策略中找显著 → FDR 控制
- Bonferroni: p_adj = p * N (太严)
- Holm-Bonferroni: 阶梯式
- BH (Benjamini-Hochberg): 控制 FDR
"""
import numpy as np
import pandas as pd
from statsmodels.stats.multitest import multipletests


def multipletest_correction(p_values: list, method: str = 'holm') -> pd.DataFrame:
    reject, p_adj, _, _ = multipletests(p_values, method=method, alpha=0.05)
    return pd.DataFrame({
        'p_raw': p_values,
        'p_adj': p_adj,
        'reject': reject
    })
```
