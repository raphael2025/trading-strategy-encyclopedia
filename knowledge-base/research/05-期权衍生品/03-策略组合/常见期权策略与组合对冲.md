## 一、方向性策略

### 1. 买入看涨（Long Call）

**适用场景**：强烈看涨标的

**构造**：买入 1 份 Call
- 最大亏损：权利金
- 最大盈利：∞
- 盈亏平衡：行权价 + 权利金

**风险/收益**：
- ✅ 风险有限、潜在收益无限
- ❌ 高时间衰减（短期 Call 损失 30-50% 时间价值）
- ❌ 需要标的显著上涨才能盈利

**案例**：BTC 现 65,000，买 1 个月 70,000 Call @ 1,500 USD
- 盈亏平衡：71,500
- BTC 涨到 80,000 → 盈利 8,500（+567%）
- BTC 维持 65,000 → 损失 1,500

**加密实战**：Deribit 上最常见的散户策略之一，2024 年"美国 BTC 战略储备"消息后短期 Call 需求激增

### 2. 买入看跌（Long Put）

**适用场景**：强烈看跌标的

**构造**：买入 1 份 Put
- 最大亏损：权利金
- 最大盈利：行权价 × 1
- 盈亏平衡：行权价 - 权利金

**风险/收益**：
- ✅ 风险有限、潜在收益大
- ✅ 标的跌也能赚
- ❌ 同样有高时间衰减

**案例**：BTC 现 65,000，买 1 个月 60,000 Put @ 1,200 USD
- 盈亏平衡：58,800
- BTC 跌到 50,000 → 盈利 8,800（+733%）
- BTC 涨到 70,000 → 损失 1,200

### 3. 牛市价差（Bull Call Spread）

**适用场景**：温和看涨，愿意限制最大收益换取低成本

**构造**：
- 买入 K1 Call（低行权价）
- 卖出 K2 Call（高行权价，K2 > K1）
- 净支出：权利金差

**案例**：BTC 现 65,000
- 买 65,000 Call @ 2,000
- 卖 70,000 Call @ 1,000
- 净支出：1,000

| BTC 价格 | 65,000 Call | 70,000 Call | 净 P&L |
|---------|-------------|-------------|--------|
| 60,000 | 0 | 0 | -1,000 |
| 65,000 | 0 | 0 | -1,000 |
| 70,000 | 5,000 | 0 | +4,000 |
| 75,000 | 10,000 | 5,000 | +4,000 |
| 80,000 | 15,000 | 10,000 | +4,000 |

- 最大亏损：1,000
- 最大盈利：4,000
- 盈亏平衡：66,000

**优势**：比直接买 Call 便宜 50%

### 4. 熊市价差（Bear Put Spread）

**适用场景**：温和看跌

**构造**：
- 买 K1 Put
- 卖 K2 Put（K2 < K1）
- 净支出：权利金差

## 二、波动率策略

### 5. 跨式（Straddle）

**适用场景**：预期大幅波动，方向不明

**构造**：
- 买入 1 ATM Call + 买入 1 ATM Put（同到期日）

**案例**：BTC 现 65,000
- 买 65,000 Call @ 2,000
- 买 65,000 Put @ 2,000
- 净支出：4,000

**盈亏平衡**：61,000 / 69,000
- BTC 涨 5% → 赚 ~2,250
- BTC 涨 10% → 赚 ~6,500
- BTC 跌 5% → 赚 ~2,250
- BTC 维持 65,000 → 损失 4,000（Theta 衰减）

**经典应用**：FOMC、CPI、ETF 决定、减半等重大事件
- 2024-01 BTC ETF 批准前：7 天 Straddle 价格飙升 200%
- 2024-05 ETH ETF 决定前：类似现象

### 6. 宽跨式（Strangle）

**适用场景**：预期大幅波动但比 Straddle 便宜

**构造**：
- 买 OTM Call（K1 > S）
- 买 OTM Put（K2 < S）

**案例**：BTC 现 65,000
- 买 70,000 Call @ 1,000
- 买 60,000 Put @ 1,000
- 净支出：2,000（比 Straddle 便宜 50%）

**盈亏平衡**：58,000 / 72,000（比 Straddle 范围更宽）
- 需要更大波动才能盈利
- 但成本低

### 7. 蝶式（Butterfly）

**适用场景**：预期标的接近某价格

**构造**（Call Butterfly）：
- 买 1 K1 Call
- 卖 2 K2 Call（K2 = (K1+K3)/2）
- 买 1 K3 Call

**最大盈利**：在 K2 实现
**最大亏损**：权利金净支出

**案例**：BTC 现 65,000，预期 30 天后还在 65,000 附近
- 买 60,000 Call @ 6,000
- 卖 2 张 65,000 Call @ 3,000×2 = 6,000
- 买 70,000 Call @ 1,000
- 净支出：1,000

| BTC 价格 | 价值 | 净 P&L |
|---------|------|--------|
| 60,000 | 0 | -1,000 |
| 65,000 | 0 (短 2 张 65,000 Call 行权) | +4,000 |
| 70,000 | 0 (Call 价 0) | -1,000 |

- 最大盈利 4,000（BTC 正好 65,000）
- 最大亏损 1,000

**优势**：成本极低 + 明确收益

### 8. 铁鹰（Iron Condor）

**适用场景**：预期标的横盘，胜率高

**构造**：
- 卖 OTM Put（K1）
- 买更 OTM Put（K2，K2 < K1）
- 卖 OTM Call（K3）
- 买更 OTM Call（K4，K4 > K3）

**最大盈利**：所有权利金收入
**最大亏损**：行权价间距 - 净收入

**案例**：BTC 现 65,000
- 卖 60,000 Put @ 1,000
- 买 58,000 Put @ 500
- 卖 70,000 Call @ 1,000
- 买 72,000 Call @ 500
- 净收入：1,000

| BTC 价格 | 净 P&L |
|---------|--------|
| 56,000 | -1,000 (最大亏损) |
| 58,000 | 0 |
| 60,000 | 0 |
| 65,000 | +1,000 (最大盈利) |
| 70,000 | 0 |
| 72,000 | -1,000 (最大亏损) |

- 胜率 60-80%
- 适合波动率高的市场
- 加密常用：周末 / 横盘期

## 三、收益增强策略

### 9. Covered Call

**适用场景**：长期持有标的，希望降本

**构造**：
- 持有 1 份标的
- 卖出 1 份 OTM Call

**风险/收益**：
- 赚权利金
- 限制上方收益
- 保留下行风险（除非对冲）

**案例**：持有 1 BTC（@ 65,000），卖 70,000 Call（30 天）@ 1,000
- 30 天后 BTC = 68,000：Call 归零，净赚 1,000
- 30 天后 BTC = 75,000：BTC 赚 10,000 - Call 行权（70,000）= 净赚 6,000（+ 1,000 权利金 = 7,000）
- 30 天后 BTC = 60,000：BTC 亏 5,000 + 权利金 1,000 = 净亏 4,000

**加密实战**：Deribit、OKX 长期持有人常用

### 10. Cash-Secured Put

**适用场景**：愿意在更低价格接货

**构造**：
- 持有等额现金
- 卖出 OTM Put

**收益**：权利金
**风险**：被行权（必须按行权价买入）

**案例**：BTC 现 65,000，卖 60,000 Put（30 天）@ 1,000 USD
- 30 天后 BTC > 60,000：Put 归零，净赚 1,000
- 30 天后 BTC = 55,000：被行权，必须 60,000 买入 BTC（"接刀"）

## 四、对冲策略

### 11. Protective Put（保护性看跌）

**适用场景**：持有现货 + 担心下跌

**构造**：
- 持有 1 份标的
- 买入 OTM Put

**效果**：
- 下行保护：到行权价为止
- 上行无限制
- **成本**：Put 权利金

**案例**：持有 1 BTC（@ 65,000），买 60,000 Put（30 天）@ 1,200
- BTC 跌到 50,000：现货亏 15,000 + Put 赚 9,000 = 净亏 6,000
- BTC 涨到 80,000：现货赚 15,000 - Put 权利金 1,200 = 净赚 13,800

**对比 Covered Call**：
- Protective Put：付权利金、保留上行（保险思维）
- Covered Call：收权利金、限制上行（收益思维）

### 12. Collar（领子策略）

**适用场景**：长期持有 + 控制风险 + 愿意限制收益

**构造**：
- 持有标的
- 买 OTM Put（保护下行）
- 卖 OTM Call（限制上行）

**特征**：
- 成本：通常 Call 权利金 - Put 权利金 = 净收或小付
- 下行保护：到 Put 行权价
- 上行限制：到 Call 行权价

**案例**：持有 1 BTC（@ 65,000）
- 买 60,000 Put @ 1,200
- 卖 70,000 Call @ 1,000
- 净成本：200

- BTC 跌到 55,000：现货亏 10,000 - Put 行权 +10,000 = 0，再减 200 净成本 = 净亏 200
- BTC 涨到 75,000：现货赚 10,000 - Call 行权（70,000）= 净赚 0，再减 200 净成本 = 净亏 200
- BTC 维持 65,000：净亏 200

**优势**：几乎零成本建立保护
**劣势**：错失大幅上涨

**MicroStrategy 真实案例**：2024 年用 Collar 策略对冲 200K+ BTC 持仓

## 五、波动率与方向组合

### 13. Calendar Spread

**适用场景**：预期短期 IV > 长期 IV（Term Structure）

**构造**：
- 卖近月 ATM 期权
- 买远月 ATM 期权（同 Strike）

**特征**：
- 短期赚 Theta
- 长期 Vega 暴露
- 风险：单边大幅波动

### 14. Diagonal Spread

**适用场景**：用远期对冲 + 赚时间

**构造**：
- 卖近月 OTM Call
- 买远月 ITM Call
- 净支出：权利金差

## 六、加密特色策略

### 15. Funding Rate Hedge（资金费率对冲）

**场景**：永续合约资金费率高，做空永续 + 买入现货

**构造**：
- 买入 1 BTC 现货
- 卖出 1 BTC 永续合约（永续对冲）
- 净赚 Funding Rate 收益

**收益**：Funding Rate 0.05%/天 × 3 = 0.15%/周
**风险**：永续与现货价格偏离

### 16. Basis Trade（基差套利）

**场景**：现货 - 期货基差异常

**构造**：
- 买入现货
- 卖出期货合约（季度）
- 持有到期交割
- 赚基差收敛

**典型收益**：年化 5-15%
**风险**：交割失败、流动性

### 17. Gamma Scalping（Gamma 剥头皮）

**场景**：持有长 Gamma 头寸（如 Short Straddle），主动 Delta 对冲

**操作**：
- 持有 Short Straddle
- BTC 涨 → 卖出 BTC 现货
- BTC 跌 → 买入 BTC 现货
- 赚每次波动的 0.1-0.5%

**加密实战**：Jump、Cumberland 等做市商的核心策略

### 18. Skew Trade（偏斜交易）

**场景**：预期 Put 偏斜（Overskewed）缩小

**构造**：
- 卖 OTM Call（低 IV）
- 买 OTM Put（高 IV）
- 赌 Put/Call IV 差缩小

## 九、关键经验教训

### 1. 散户常见错误

- **过度交易**：每天买 Call，Theta 吃掉所有利润
- **远期虚值 Call**：90% 概率归零
- **卖出裸期权**：一次爆亏前面所有利润
- **忽略波动率**：高 IV 环境下卖期权 → 黑天鹅
- **混合策略混乱**：买了 Put 又卖 Call，结果像裸卖

### 2. 风险管理

- 单笔策略风险 ≤ 账户 5%
- 永远理解最大亏损
- 卖出期权需要保证金（通常 10-30% 名义价值）
- 周末降低 delta 暴露
- 重大事件前主动减仓

### 3. 心理建设

- 接受 60-70% 胜率（不是 100%）
- Iron Condor 80% 胜率但最大亏损是盈利的 4 倍
- 期权交易是"长期概率游戏"

## A.1 Payoff 通用计算器

```python
"""
统一 Payoff 计算：支持多腿组合
- leg: dict {type, position, strike, premium}
- spot_prices: np.ndarray of 标的价格
"""
import numpy as np
import pandas as pd


def leg_payoff(leg: dict, spots: np.ndarray) -> np.ndarray:
    """
    leg: {'type': 'call'|'put', 'position': 'long'|'short',
          'strike': K, 'premium': p}
    spots: 标的价格数组
    返回到期日 payoff（不含初始权利金）
    """
    K = leg['strike']
    if leg['type'] == 'call':
        intrinsic = np.maximum(spots - K, 0)
    else:
        intrinsic = np.maximum(K - spots, 0)
    if leg['position'] == 'short':
        return -intrinsic
    return intrinsic


def strategy_payoff(legs: list[dict], spots: np.ndarray,
                    include_premium: bool = True) -> np.ndarray:
    """组合 payoff = Σ(leg_payoff) - Σ(short 收入 - long 付出) 权利金"""
    payoff = sum(leg_payoff(l, spots) for l in legs)
    if include_premium:
        net_premium = sum(
            (-l['premium'] if l['position'] == 'long' else l['premium'])
            for l in legs
        )
        payoff = payoff + net_premium
    return payoff
```

## A.2 四类基础策略 Payoff 模板

```python
"""
四个加密最常用策略：
- Covered Call（持币 + 卖 OTM Call）
- Protective Put（持币 + 买 Put 保险）
- Bull Call Spread（买低 Call + 卖高 Call）
- Iron Condor（卖 OTM Put + 买更低 Put + 卖 OTM Call + 买更高 Call）
"""
import numpy as np
from A1 import strategy_payoff


def covered_call(spot0: float, strike_call: float, premium_call: float,
                 spots: np.ndarray) -> np.ndarray:
    """持币 + 卖 Call（收入权利金但放弃 > strike 的涨幅）"""
    return spots - spot0 + strategy_payoff(
        [{'type': 'call', 'position': 'short', 'strike': strike_call,
          'premium': premium_call}],
        spots, include_premium=True)


def protective_put(spot0: float, strike_put: float, premium_put: float,
                   spots: np.ndarray) -> np.ndarray:
    """持币 + 买 Put（保险，下方 S 跌至 K 不再亏）"""
    return (spots - spot0) + strategy_payoff(
        [{'type': 'put', 'position': 'long', 'strike': strike_put,
          'premium': premium_put}],
        spots, include_premium=True)


def bull_call_spread(strike_long: float, strike_short: float,
                     prem_long: float, prem_short: float,
                     spots: np.ndarray) -> np.ndarray:
    """买低 call 卖高 call，净权利金 = prem_long - prem_short"""
    legs = [
        {'type': 'call', 'position': 'long', 'strike': strike_long, 'premium': prem_long},
        {'type': 'call', 'position': 'short', 'strike': strike_short, 'premium': prem_short},
    ]
    return strategy_payoff(legs, spots)


def iron_condor(put_long: float, put_short: float, call_short: float, call_long: float,
                prem_pl: float, prem_ps: float, prem_cs: float, prem_cl: float,
                spots: np.ndarray) -> np.ndarray:
    """卖宽跨式：低 put spread + 高 call spread"""
    legs = [
        {'type': 'put', 'position': 'long', 'strike': put_long, 'premium': prem_pl},
        {'type': 'put', 'position': 'short', 'strike': put_short, 'premium': prem_ps},
        {'type': 'call', 'position': 'short', 'strike': call_short, 'premium': prem_cs},
        {'type': 'call', 'position': 'long', 'strike': call_long, 'premium': prem_cl},
    ]
    return strategy_payoff(legs, spots)
```

## A.3 动态 Delta 对冲（离散再平衡）

```python
"""
模拟：每天调仓使组合 Delta = 0（用 BS 计算 Delta）
- 简化：每个 step 取一天；手续费按 0.04% per side
"""
import numpy as np
import pandas as pd
from A1 import bs_price  # 同 05-01 附录
from A2 import bs_greeks  # 同 05-01 附录


def delta_hedge_simulation(spot_path: np.ndarray, K: float, T_initial: float,
                            r: float, sigma: float, freq: int = 1,
                            fee_rate: float = 0.0004,
                            option_type: str = "short_call") -> pd.DataFrame:
    """
    spot_path: 标的价格日序列
    option_type: 卖 Call/卖 Put 等
    返回：每日 portfolio_value（含累计对冲盈亏 + 期权 MTM）
    """
    n = len(spot_path)
    pos = option_type.split("_")[0]   # 'short' or 'long'
    typ = option_type.split("_")[1]   # 'call' or 'put'

    inventory = 0  # 正数 = 持币
    cash = bs_price(spot_path[0], K, T_initial, r, sigma, typ) * (-1 if pos == "short" else 1)
    history = []

    for i in range(n):
        S = spot_path[i]
        T = max(T_initial - i / 365, 1e-9)
        delta = bs_greeks(S, K, T, r, sigma, typ)["delta"]
        if pos == "short":
            delta = -delta
        # Rebalance: 持有 -delta 单位的标的（对冲掉期权 delta 暴露）
        target_inventory = -delta
        trade = target_inventory - inventory
        cash -= trade * S
        cash -= abs(trade) * S * fee_rate   # 手续费
        inventory = target_inventory

        option_mtm = bs_price(S, K, T, r, sigma, typ) * (-1 if pos == "short" else 1)
        portfolio = cash + inventory * S + option_mtm
        history.append({
            "day": i, "spot": S, "delta": delta,
            "inventory": inventory, "cash": cash,
            "option_mtm": option_mtm, "portfolio": portfolio
        })
    return pd.DataFrame(history)
```

## A.4 组合 Greeks 聚合（多腿求和）

```python
"""
组合 Greeks = Σ 单腿 Greeks × 手数
- 长腿 delta 为正，short delta 为负
- 用于 portfolio-level 风险监控
"""
import pandas as pd
from A1 import bs_greeks


def portfolio_greeks(positions: list[dict], S: float, r: float = 0.05) -> dict:
    """
    positions: [{'type': 'call', 'position': 'long'|'short', 'strike': K,
                 'T': 0.1, 'sigma': 0.6, 'qty': 1, 'contract_size': 0.1}, ...]
    S: 标的价格（所有腿标的相同）
    返回: dict of aggregated greeks
    """
    greeks_sum = {"delta": 0, "gamma": 0, "vega": 0, "theta": 0, "rho": 0}
    for p in positions:
        g = bs_greeks(S, p['strike'], p['T'], r, p['sigma'], p['type'])
        sign = 1 if p['position'] == 'long' else -1
        mult = sign * p['qty'] * p.get('contract_size', 1)
        for k in greeks_sum:
            greeks_sum[k] += g[k] * mult
    return greeks_sum
```

## A.5 期权组合回测框架（事件驱动简化版）

```python
"""
事件驱动回测：每根 bar 重新 mark-to-market，记录组合价值
简化：每天一次结算，暂不处理分钟级 slippage
"""
import numpy as np
import pandas as pd
from A1 import bs_price


def backtest_option_portfolio(spot_df: pd.DataFrame, leg_specs: list[dict],
                              fee_rate: float = 0.0004) -> pd.DataFrame:
    """
    spot_df: OHLCV DataFrame
    leg_specs: 每条腿 {'entry_day', 'type', 'position', 'strike', 'T_days', 'sigma', 'qty'}
    返回净值曲线
    """
    portfolio = pd.DataFrame(index=spot_df.index, columns=["value"], data=0.0)
    open_legs = []
    for _, row in spot_df.iterrows():
        # 开仓
        for spec in leg_specs:
            if pd.Timestamp(spec['entry_day']) == row.name:
                entry_price = bs_price(row['close'], spec['strike'],
                                       spec['T_days']/365, 0.05, spec['sigma'],
                                       spec['type'])
                sign = 1 if spec['position'] == 'long' else -1
                portfolio.loc[row.name, 'value'] -= sign * entry_price * spec['qty'] \
                                                    * (1 + fee_rate)
                open_legs.append({**spec, 'entry_price': entry_price})
        # 持仓估值
        val = 0
        for leg in open_legs:
            t_remaining = max(leg['T_days']/365 - (row.name.dayofyear - pd.Timestamp(leg['entry_day']).dayofyear)/365, 0)
            if t_remaining > 0:
                mtm = bs_price(row['close'], leg['strike'], t_remaining, 0.05,
                               leg['sigma'], leg['type'])
                sign = 1 if leg['position'] == 'long' else -1
                val += sign * mtm * leg['qty']
        portfolio.loc[row.name, 'value'] = val
    return portfolio
```

## A.6 加密期权 IV-RV 套利回测（VRP）

```python
"""
VRP 套利 = 卖期权（IV） + 动态 Delta 对冲（实现波动率 RV）
- 持续卖出 30-45 DTE ATM Straddle
- 每日对冲 delta
- VRP > 0 时收益
"""
import numpy as np
import pandas as pd
from A3 import delta_hedge_simulation


def vrp_backtest(spot: pd.Series, iv: pd.Series, rv_30d: pd.Series,
                 fee_rate: float = 0.0004, threshold: float = 0.10) -> pd.DataFrame:
    """
    spot: 标的价格日序列
    iv: 30 天 ATM 隐含波动率（同期）
    rv_30d: 30 天已实现波动率（同期）
    threshold: VRP 触发阈值
    """
    vrp = iv - rv_30d
    pnl = []
    for i in range(len(spot)):
        if vrp.iloc[i] > threshold:
            # 卖 30D ATM Straddle → 卖 call + 卖 put
            sigma = iv.iloc[i]
            # 简化：用 call 单腿代表（实际应双腿）
            sim = delta_hedge_simulation(
                spot.iloc[max(0, i-30):i+1].values,
                K=spot.iloc[i], T_initial=30/365, r=0.05, sigma=sigma,
                freq=1, fee_rate=fee_rate, option_type="short_call"
            )
            pnl.append(sim['portfolio'].iloc[-1])
        else:
            pnl.append(0)
    return pd.DataFrame({"date": spot.index, "pnl": pnl})
```
