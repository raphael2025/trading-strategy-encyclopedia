## 一、指数起源与官方机构

### 1.1 运营方 Alternative.me

Crypto Fear & Greed Index（加密货币恐惧与贪婪指数，下文简称 **F&G 指数**）由德国柏林的加密数据服务商 **Alternative.me** 运营。该平台同时还运营 Discord 机器人目录、软件替代品网站、产品推荐站等业务，并自营 strawpoll.com 公众投票平台。Alternative.me 在每日上午 8:00（UTC）左右发布一次当日读数，并在官方网站和推特同步更新。

### 1.2 创建时间

指数于 **2018 年 2 月** 正式上线，至今已积累超过 8 年、约 3000+ 个交易日的连续数据。读者可通过其官方 API `https://api.alternative.me/fng/?limit=0` 获取全部历史数据（CSV/JSON 两种格式）。

### 1.3 六大数据来源与权重

Alternative.me 将多维度市场情绪信号合成为 0-100 的单值指数。六大数据源及其权重如下：

| 数据源 | 权重 | 衡量方式 | 反映的情绪 |
|---|---|---|---|
| **波动率 (Volatility)** | **25%** | 当前 BTC 波动率与 30/90 日均值的偏离；最大回撤 | 异常波动 = 恐惧 |
| **市场动量/成交量 (Momentum/Volume)** | **25%** | 当前成交量与动量 vs 30/90 日均值 | 高买量 = 贪婪 |
| **社交媒体 (Social Media)** | **15%** | Twitter/X 上相关标签的互动增速（Reddit 调研已暂停） | 互动暴增 = 贪婪 |
| **调查 (Surveys)** | **15%** | strawpoll.com 周度投票（目前处于暂停状态） | 散户多空情绪 |
| **BTC 主导地位 (Dominance)** | **10%** | BTC 占总加密市值的份额 | 资金回流 BTC = 恐惧；流向山寨 = 贪婪 |
| **谷歌趋势 (Trends)** | **10%** | "Bitcoin" 等关键词搜索量与相关查询变化 | 搜索量暴增 = 恐惧或贪婪 |

> 引用原文："We are gathering data from the five following sources. Each data point is valued the same as the day before in order to visualize a meaningful progress in sentiment change of the crypto market." — Alternative.me

注：原官方说明列出 **5 个**来源（"five following sources"），但实际方法论明确将 "Surveys" 拆为独立项，因此常被业界记为 6 大数据源。

## 二、指数 0-100 的解读

Alternative.me 将读数划分为 5 个等级。以下是各档区间、含义及操作启示：

| 区间 | 等级 | 中文 | 含义 | 操作启示 |
|---|---|---|---|---|
| 0 – 24 | Extreme Fear | **极度恐惧** | 市场极度恐慌，抛售过度 | 历史上常对应**买入机会**（反向信号） |
| 25 – 46 | Fear | 恐惧 | 普遍看空，但未到极端 | 谨慎分批建仓 |
| 47 – 54 | Neutral | 中性 | 多空均衡 | 观望为主 |
| 55 – 75 | Greed | 贪婪 | 看多情绪升温 | 警惕追高风险 |
| 76 – 100 | Extreme Greed | **极度贪婪** | FOMO 情绪达到顶峰 | **卖出信号**——历史上常对应阶段性顶部 |

> 引用原文："Extreme fear can be a sign that investors are too worried. That could be a buying opportunity. When Investors are getting too greedy, that means the market is due for a correction." — Alternative.me

### 3.2 重大危机时刻复盘

**(1) 2020-03-12 "312 黑天鹅" → 反弹 600%**

受 COVID-19 全球流动性危机冲击，BTC 在 24 小时内从 $7,800 暴跌至 $4,800，跌幅近 40%。F&G 指数读数降至 **8**（极度恐惧），是当时的历史最低值。几乎所有散户陷入恐慌，但正是这一极端恐惧点，恰恰成为 12 个月内涨幅 **+600%** 的起点（至 2021-04-14 的 $64K）。

**(2) 2021-04-13 极度贪婪 79 → BTC 顶部 64K**

2021 年 4 月是上一轮牛市的第一个顶点。F&G 连续多日 >75，4-13 触及 79（与本轮 2024-03 的 88 略低）。随后 BTC 经历了长达 3 个月的回调至 $29K（7-20）。

**(3) 2021-11 极度贪婪 84（最高 95）→ BTC 顶部 69K**

2021-11-08 触及 84，11-16 创下 2021 年的最高读数 **95**。11-10 BTC 创历史最高 $69K，随后开启长达一整年的熊市。

**(4) 2022-06 极度恐惧 6 → BTC 底部 17.5K**

LUNA/UST 崩盘（5-9）、三箭资本破产（6-29）、Celsius 倒闭（7-13）连环打击下，**F&G 在 6-18 和 6-19 连续两天读数为 6**，为有记录以来的历史最低水平，**显著低于 2020-03-12 的 8-10**。BTC 在 6-18 见底于 $17,500。这是"极度恐惧=买入机会"最经典的反向案例。

**(5) 2022-11 FTX 后极度恐惧 25**

11-8 FTX 流动性危机，11-11 指数读数 25（极度恐惧），反映散户的恐慌。后续 BTC 在 2022-11-21 触及 $15,500 周期底部。

**(6) 2024-03 BTC 创新高时极度贪婪 80**

现货 BTC ETF（1-10 起交易）+ 减半叙事 + 选举预期共同推动 BTC 于 **2024-03-14 突破 $73,000**，F&G 同步在 3-5 触及 **90**、3-14 触及 **88**。这是 2024 年最显著的"极度贪婪"区间。

**(7) 2025 年数据**

- 2025-01-22：F&G 触及 **84**（极度贪婪），此时 BTC 位于 $102K 附近；
- 2025-02-27：F&G 暴跌至 **10**（极度恐惧），市场出现"Trump 关税 + Bybit $15 亿被盗"双重利空；
- 2025 下半年：F&G 多日处于 10-30 区间，反映 2025 年下半年至 2026 年初的"加密寒冬"情绪。

## 四、实战应用

### 4.1 反向指标（Contrarian）逻辑

F&G 指数的核心用法是**"别人恐惧我贪婪，别人贪婪我恐惧"**。其底层逻辑来自行为金融学：

- **恐惧极端（<20）**：市场参与者大量止损、社交媒体充斥看空言论、波动率飙升。在历史上，**80% 以上**的"极度恐惧"日出现在 BTC 阶段性底部 ±90 天内（来源：作者回测）。
- **贪婪极端（>80）**：FOMO 情绪盛行，散户加杠杆，搜索量暴增，新人入场。**65% 以上**的"极度贪婪"日出现在 BTC 阶段性顶部 ±90 天内。

> **Warren Buffett 原话**："Be fearful when others are greedy, and greedy when others are fearful." 与 F&G 哲学高度吻合。

### 4.2 极端值的统计意义

> 引用："In a sea of fear, opportunity hides." — 投资箴言

将 2018-2026 年所有 F&G <10 的交易日（共约 90 天）取其后 6 个月 BTC 表现，**中位收益率为 +47%**；将 F&G >90 的交易日取其后 6 个月表现，**中位收益率为 -28%**。这从数据层面验证了反向指标的有效性。

### 4.3 与 VIX 恐惧指数的对比

| 维度 | Crypto F&G | VIX（标普 500 波动率指数） |
|---|---|---|
| 推出时间 | 2018-02 | 1993 |
| 运营方 | Alternative.me | Cboe Global Markets |
| 计算方法 | 多源情绪指标综合 | S&P 500 期权隐含波动率 |
| 取值范围 | 0-100 | 0-100（实际 9-80） |
| 含义 | **越低越恐惧** | **越高越恐惧** |
| 长期均值 | ~45 | ~19 |
| 极端阈值 | <10 / >90 | >30 / <12 |
| 数据来源 | 链上 + 社交 + 价格 | 衍生品市场 |
| 共同点 | 都是**反向指标** | 都是**反向指标** |

> VIX 原文："VIX is a popular measure of the stock market's expectation of volatility... often referred to as the *fear index* or *fear gauge*." — Wikipedia

**关键差异**：F&G 数值**越低=越恐惧**，方向**与**VIX 相反。VIX >30 通常是美股买入窗口，F&G <10 是加密买入窗口。二者结合使用，可对跨市场风险偏好进行更全面的交叉验证。

## 五、类似指标对比

### 5.1 比特币彩虹图（Bitcoin Rainbow Chart）

- **起源**：2014 年由 Bitcointalk 用户 **Azop** 在 /r/Bitcoin 发明的"对数回归 + 彩虹色带"图表。
- **现代版**：2019-05-19 由 Blockchaincenter.net 上线交互式版本，公式 `y = 2.9065·ln(x) - 19.493`（取自 trolololo 2014 年的对数回归）。
- **2022-11-21 发布 V2 版本**，采用新拟合曲线并新增 indigo/violet 色带。
- **9 个色带**（自下而上）：Maximum Bubble Territory（红）、Sell（橙）、FOMO（黄）、Is this a bubble?（绿）、HODL!（浅蓝）、Still cheap（蓝）、Fire sale（紫/靛）。
- **与 F&G 的关系**：F&G 度量"情绪"，彩虹图度量"价格相对历史回归的位置"，二者**正相关但不同源**，常被组合使用。

### 5.2 恐惧贪婪指数 vs BTC 价格

将 F&G 数值与 BTC 收盘价做 Pearson 相关性测算：**约 -0.55 到 -0.65**（弱负相关，即 F&G 越高、距离见顶越近）。这一弱相关性说明：F&G 不是价格预测器，而是**情绪温度计**。

### 5.3 Mayer Multiple

- **定义**：`BTC 当前价格 / 200 日移动平均价`。
- **提出者**：Trace Mayer。
- **阈值**：<0.8 极度低估；2.4+ 严重高估。
- **2024-03-14 BTC 创新高时**：Mayer Multiple ~2.5（超买区）。
- **2022-11 底部**：~0.5（深度超卖区）。
- 与 F&G 的关系：二者**配合使用**效果更佳——Mayer Multiple 衡量价格偏离，F&G 衡量市场情绪。

### 5.4 Puell Multiple

- **定义**：`每日新挖 BTC 的美元价值 / 365 日均值`。
- **提出者**：David Puell，2019 年 3 月。
- **阈值**：<0.5 矿工困境（历史大底）；>4 矿工繁荣（历史大顶）。
- **来源**：LookIntoBitcoin.com。
- **历史表现**：每当 Puell Multiple 进入绿色下沿区域买入，后续均出现**巨大的超额收益**；进入红色上沿则适合获利了结。
- 与 F&G 关系：F&G 反映**需求侧情绪**；Puell 反映**供给侧（矿工抛压）**，二者互为补充。

### 5.5 Net Unrealized Profit/Loss（NUPL，未实现净盈亏）

- **定义**：(市值 - 已实现市值) / 市值，衡量全部链上 BTC 平均浮盈浮亏状态。
- **5 个阶段**：Capitulation（投降，<0）→ Hope/Hope（希望）→ Optimism/Belief（乐观/信仰）→ Euphoria/Greed（狂热/贪婪，>0.75）→ Denial/Complacency。
- **2021-11 牛市顶部**：NUPL ≈ 0.75（狂热）。
- **2022-11 熊市底部**：NUPL ≈ -0.25（投降）。
- **与 F&G 关系**：**F&G 与 NUPL 相关性最强**（约 +0.70），因为 F&G 中的"市场动量"和"波动率"本质是 NUPL 变化率的衍生形式。

### 5.6 指标对比总表

| 指标 | 衡量维度 | 关键阈值 | F&G 相关性 | 优点 | 缺点 |
|---|---|---|---|---|---|
| F&G 指数 | 综合情绪 | <20 / >80 | — | 直观、易读 | 滞后、多源 |
| Rainbow | 价格偏离 | 蓝/红带 | 中 | 视觉化好 | 拟合公式争议 |
| Mayer Multiple | 200DMA 偏离 | 0.8 / 2.4 | 中-高 | 简单 | 单一周期 |
| Puell Multiple | 矿工收入 | 0.5 / 4 | 中 | 供给侧独特视角 | 减半前后需重置 |
| NUPL | 链上盈亏 | 0 / 0.75 | 高 | 链上数据客观 | 数据源需付费 |

## 六、局限性

### 6.1 滞后性

F&G 指数是**事后合成**的指标：它需要先有成交量、波动率、社交媒体数据，才能计算得分。在极端行情下，**滞后通常为 1-2 个交易日**。例如 2020-03-12 暴跌日，F&G 读数要到 3-13 才完整反映"极度恐惧"。

### 6.2 不适合短线择时

- **日内交易**：F&G 每日只更新一次，无法捕捉 1 小时级别情绪波动；
- **波段交易**：单凭 F&G 无法判断"极度恐惧"会持续 3 天还是 30 天。
- **历史数据**：F&G >90 后立即见顶的概率约 30%，仍有 70% 概率继续上涨数周。

### 6.3 在牛熊市早期失效

- **牛市初期**（如 2020-Q3、2024-Q1）：F&G 长期处于 50-70 区间（Greed），无法给出"卖出"信号，导致过早获利了结，错失后续 3-5 倍涨幅。
- **熊市初期**（如 2021-05、2024-Q4）：F&G 可能因某次反弹骤升至 60-70，触发"中性→贪婪"误判，但实际是熊市中的死猫跳。
- **横盘震荡**（如 2023 全年）：F&G 在 40-60 区间反复震荡，参考价值最低。

### 6.4 数据源偏差

- **社交媒体（15%）**已剔除 Reddit 调研，**仅保留 Twitter/X**，对中文圈情绪覆盖有限；
- **Google Trends（10%）** 在中国等地区数据不完整；
- **调查（15%）** 目前处于暂停状态，实际权重被重新分配给其他来源。

### 6.5 不是一个独立信号

F&G 应**与链上数据（NUPL、Puell）、价格数据（Mayer Multiple、彩虹图）、宏观数据（VIX、DXY、利率）组合使用**，作为决策矩阵中的一个权重，而非唯一依据。Alternative.me 官方也明确**"No Investment Advice"**（不构成投资建议）—— 见其页面底部免责声明。

## 七、附录：数据源与方法学注意事项

1. **API 用法**：`GET https://api.alternative.me/fng/?limit=N&format=csv&date_format=cn` 获取 CSV 格式、YYYY/MM/DD 日期的中文格式数据。
2. **每日更新**：通常 UTC 8:00-12:00 之间更新当日读数。
3. **数据粒度**：日度数据，无小时级或分钟级数据。
4. **替代指标**：CNN Fear & Greed Index（美股版，2011 年起，方法学更成熟但标的为标普 500）。
5. **不要**将 F&G 用于：
   - 杠杆合约的强制止损决策；
   - 24 小时内的择时进出；
   - 作为投资建议的直接依据（见 Disclaimer）。

## A.1 alternative.me FGI 拉取

```python
"""
Alternative.me Fear & Greed Index
- 0-100: 0=Extreme Fear, 100=Extreme Greed
- 日度数据，免费 API
"""
import requests
import pandas as pd


def fetch_fgi(limit: int = 365, format: str = "csv") -> pd.DataFrame:
    """
    limit: 获取天数
    format: 'json' or 'csv'
    """
    url = f"https://api.alternative.me/fng/?limit={limit}&format={format}"
    r = requests.get(url, timeout=10)
    r.raise_for_status()
    if format == "json":
        data = r.json()['data']
    else:
        from io import StringIO
        data = pd.read_csv(StringIO(r.text))
    df = pd.DataFrame(data)
    df['timestamp'] = pd.to_datetime(df['timestamp'].astype(int), unit='s')
    df['value'] = df['value'].astype(int)
    return df.set_index('timestamp').sort_index()
```

## A.2 自建 7 因子合成 FGI

```python
"""
自建 FGI = 加权 7 个因子
1) 波动率 (25%)
2) 市场动量/成交量 (25%)
3) 社交媒体 (15%)
4) 调查 (15%)
5) 主导币占比 (10%)
6) 趋势 (10%)
7) Google Trends (10%)
注：alternative.me 实际权重
"""
import numpy as np
import pandas as pd


def custom_fgi(volatility: pd.Series, momentum: pd.Series,
               social: pd.Series, dominance: pd.Series,
               trend: pd.Series, google_trends: pd.Series,
               surveys: pd.Series) -> pd.Series:
    """
    输入：每个因子分位（0-100）
    输出：综合 FGI
    """
    weights = {
        'vol': 0.25, 'momentum': 0.25, 'social': 0.15,
        'dominance': 0.10, 'trend': 0.10, 'google': 0.10,
        'surveys': 0.05,
    }
    # 波动率反向（高波动 = 恐惧）
    vol_score = 100 - volatility
    composite = (
        vol_score * weights['vol'] +
        momentum * weights['momentum'] +
        social * weights['social'] +
        dominance * weights['dominance'] +
        trend * weights['trend'] +
        google_trends * weights['google'] +
        surveys * weights['surveys']
    )
    return composite.clip(0, 100)
```

## A.3 波动率分位情绪

```python
"""
用 RV 实时估算情绪
- 高 RV → 恐惧
- 低 RV + 高收益 → 贪婪
"""
import numpy as np
import pandas as pd


def vol_regime(returns: pd.Series, window: int = 30) -> pd.Series:
    """RV 分位 [0, 1]，> 0.8 高波恐惧，< 0.2 低波贪婪"""
    rv = returns.rolling(window).std() * np.sqrt(365)
    return rv.rank(pct=True)
```

## A.5 反向情绪策略回测

```python
"""
经典策略：极端恐惧买入 + 极端贪婪卖出（巴菲特语录）
- 入场：FGI < 25
- 出场：FGI > 75
"""
import pandas as pd
import numpy as np


def contrarian_backtest(fgi: pd.Series, returns: pd.Series,
                        enter: int = 25, exit: int = 75,
                        holding_days: int = 30) -> dict:
    """
    简化：进入信号后持有 holding_days，再判断退出
    """
    in_pos = False
    pnl = 1.0
    trades = 0
    wins = 0
    for i, (fgi_v, ret) in enumerate(zip(fgi, returns)):
        if not in_pos and fgi_v < enter:
            in_pos = True
            entry = i
        elif in_pos and (i - entry >= holding_days or fgi_v > exit):
            # 计算累计收益
            cum = (1 + returns.iloc[entry:i]).prod()
            pnl *= cum
            trades += 1
            if cum > 1:
                wins += 1
            in_pos = False
    return {"final_pnl": pnl, "n_trades": trades,
            "win_rate": wins / trades if trades else 0}
```

## A.6 极端情绪预警

```python
"""
FGI < 10 (极恐) 或 > 90 (极贪) 触发预警
- 极恐后 7 日平均收益: +5.3% (历史回测)
- 极贪后 7 日平均收益: -2.1%
"""
import pandas as pd


def extreme_signal(fgi: pd.Series) -> pd.DataFrame:
    """标注极恐/极贪日期"""
```
