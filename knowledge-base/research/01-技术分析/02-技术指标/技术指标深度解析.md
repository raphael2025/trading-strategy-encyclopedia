## 1. 趋势指标

趋势指标用于识别和跟随市场方向。当市场处于明确的上涨或下跌趋势时，趋势指标能够提供高胜率的入场/出场信号；但在盘整行情中容易产生大量假信号。

### 1.1 移动平均线 (Moving Average, MA)

#### 计算原理

**简单移动平均 (SMA)**：对最近 N 个周期收盘价做算术平均。

```python
SMA_n = (P_1 + P_2 + ... + P_n) / n
```

**指数移动平均 (EMA)**：对历史价格赋予指数递减的权重，赋予近期价格更高权重。

```python
# 平滑系数
alpha = 2 / (n + 1)
# 递推公式
EMA_today = alpha * Price_today + (1 - alpha) * EMA_yesterday
```

#### SMA 与 EMA 的本质差异

| 维度 | SMA | EMA |
|------|-----|-----|
| 权重分配 | 等权重 (1/n) | 指数递减 (近期更高权重) |
| 滞后性 | 大 | 小 |
| 对新数据敏感度 | 低 | 高 |
| 抗噪能力 | 强 | 弱 |
| 适用场景 | 识别长趋势、支撑/阻力 | 快速捕捉拐点 |

#### 周期选择

| 周期 | 用途 | 适用市场状态 |
|------|------|-------------|
| MA(5) | 短期趋势、超短线 | 1H/4H 级别 BTC 波动 |
| MA(10) | 短线趋势 | 日内交易 |
| MA(20) | 中短期基准 | 4H-日线级别 |
| MA(50) | 中期趋势 | 波段交易 |
| MA(100) | 中长期趋势 | 1-3 个月周期 |
| MA(200) | 长期牛熊分界 | BTC 历次牛熊分水岭 |

#### 金叉与死叉

- **金叉 (Golden Cross)**：短期均线上穿长期均线，常见组合为 MA(50) 上穿 MA(200)。BTC 在 2019 年 4 月、2020 年 12 月、2024 年 10 月均出现金叉并开启大级别上涨。
- **死叉 (Death Cross)**：短期均线下穿长期均线，历史上为重要看跌信号，但加密货币市场中假信号较多。

#### 局限

- 都是滞后指标，无法预测顶部和底部。
- 在剧烈震荡行情（盘整/急涨急跌）中频繁发出假信号。
- SMA 对尖刺毛刺极度敏感，EMA 则更易被操纵。

#### 加密货币应用

由于加密货币市场 7×24 小时不间断交易，建议使用**收盘价取整点（4H、日线收盘）**而非任意时刻价格计算 MA。BTC 的 MA(200) 周线在历史上是最受关注的牛熊分水岭指标之一（来源：[Wikipedia – Moving Average](https://en.wikipedia.org/wiki/Moving_average)）。

### 1.2 指数移动平均 (EMA) 详解

#### 与 SMA 的本质差异

EMA 的递推公式实质上是一种**无限脉冲响应 (IIR) 低通滤波器**，权重系数按指数衰减但永不为零：

```latex
\text{EMA}_t = \sum_{i=0}^{\infty} (1-\alpha)^i \cdot P_{t-i} \cdot \alpha
```

由于 EMA 对最近价格给予更高权重，因此：
- 在趋势**反转初期**能更快发出信号（灵敏度高）
- 在**震荡市**中更容易被频繁假突破触发（信号噪声大）

#### 应用要点

- 短线常用 EMA(9)、EMA(21) 配合 EMA(55)
- 中线常使用 EMA(50)、EMA(200)
- 在加密货币市场，建议同时使用 SMA(200) 与 EMA(200) 互相验证

> 来源：[Wikipedia – Moving Average](https://en.wikipedia.org/wiki/Moving_average), [Investopedia – Exponential Moving Average (EMA)](https://www.investopedia.com/terms/e/ema.asp)

### 1.3 MACD (平滑异同移动平均线)

#### 计算原理

MACD 由 Gerald Appel 于 1970 年代末期发明，是一个**收敛-发散型震荡指标**，由三部分组成：

```python
# 1. 快线 (DIF) = 12 日 EMA 减 26 日 EMA
DIF = EMA(close, 12) - EMA(close, 26)

# 2. 慢线 (DEA / Signal) = DIF 的 9 日 EMA
DEA = EMA(DIF, 9)

# 3. 柱状图 (Histogram) = DIF - DEA
HIST = DIF - DEA
```

参数 (12, 26, 9) 是历史最悠久的标准参数——12 和 26 分别对应 2 周和 1 个月（早期 6 日交易周），9 是信号线的平滑窗口。

#### 核心信号

1. **金叉/死叉**：DIF 上穿 DEA 为买入信号；下穿为卖出信号
2. **零轴穿越**：DIF 由负转正表示短期 EMA 强于长期 EMA，中长期动能偏多
3. **柱状图收窄/扩张**：柱状图由负转正或由长变短是动能增强/减弱信号
4. **背离 (Divergence)**：
   - **顶背离**：价格创新高，DIF/柱状图未创新高 → 上涨动能衰竭
   - **底背离**：价格创新低，DIF/柱状图未创新低 → 下跌动能衰竭

#### 参数调整

| 周期 | 参数 | 适用场景 |
|------|------|---------|
| 短线 | (5, 35, 5) | 快速捕捉拐点 |
| 标准 | (12, 26, 9) | 默认参数，跨市场通用 |
| 中线 | (8, 17, 9) | 老式 MACD 参数 |
| 长线 | (19, 39, 9) | 降低假信号，但滞后增加 |

#### 局限

- 作为移动平均的衍生品，本质仍是**滞后指标**。
- 在盘整行情中频繁给出错误信号。
- 零轴附近的金叉死叉可靠性低。
- 在暴涨/暴跌行情中可能出现"钝化"。

#### 加密货币应用

BTC 在 2020 年 3 月、2024 年 1 月均出现 MACD 底背离后开启大级别反弹。ETH 4H 级别的 MACD 金叉常被用作短线入场依据（来源：[Wikipedia – MACD](https://en.wikipedia.org/wiki/MACD)）。

### 1.4 抛物线指标 (Parabolic SAR)

#### 计算原理

由 J. Welles Wilder 于 1978 年提出，是**趋势跟随 + 跟踪止损**一体化指标。其核心思想是"时间是敌人"——如果价格不能持续创新高/新低，应平仓。

```python
# 核心递推公式
SAR_{t+1} = SAR_t + alpha * (EP - SAR_t)
```

其中：
- `EP (Extreme Point)`：当前趋势的最高价（上涨趋势）或最低价（下跌趋势）
- `alpha`：加速因子，初始 0.02，每次创新 EP 时 +0.02，最大 0.20
- 当 SAR 与下根 K 线价格交叉时，发出趋势反转信号

#### 应用场景

- **趋势跟踪**：SAR 持续位于价格下方为多头，SAR 持续位于价格上方为空头
- **动态止损**：将 SAR 作为追踪止损位使用
- **离场信号**：SAR 反转即趋势结束信号

#### 局限

- 在**盘整行情**中产生大量"鞭打"（whipsaw）假信号
- 加速因子变化使其对近期价格极度敏感，回测中单独使用胜率仅 19%（来源：[Liberated Stock Trader 12 年回测研究](https://www.liberatedstocktrader.com/)）
- 与 Heikin Ashi 蜡烛图配合使用可大幅提高胜率（63%）

#### 加密货币应用

BTC 日线 SAR 在 2020-2021 年牛市中完美捕捉到主升浪，每次 SAR 翻红都是最后逃顶机会。加密货币建议搭配 ADX（>25）以过滤盘整假信号（来源：[Wikipedia – Parabolic SAR](https://en.wikipedia.org/wiki/Parabolic_SAR)）。

### 1.5 平均趋向指数 (ADX)

#### 计算原理

ADX 由 Wilder 在 1978 年与 RSI、Parabolic SAR 一同提出，**专门衡量趋势强度而非方向**。

```python
# 步骤 1：计算方向运动 (+DM, -DM)
UpMove = High_today - High_yesterday
DownMove = Low_yesterday - Low_today
if UpMove > DownMove and UpMove > 0: +DM = UpMove else: +DM = 0
if DownMove > UpMove and DownMove > 0: -DM = DownMove else: -DM = 0

# 步骤 2：14 日平滑
+DI = 100 * Smoothed(+DM, 14) / ATR(14)
-DI = 100 * Smoothed(-DM, 14) / ATR(14)

# 步骤 3：ADX
DX = 100 * |+DI - -DI| / (+DI + -DI)
ADX = SMA(DX, 14)  # 实际上是 14 期 Wilder 平滑
```

#### 解读

| ADX 区间 | 含义 | 应对策略 |
|---------|------|---------|
| 0-20 | 无趋势 / 盘整 | 震荡策略，慎用趋势指标 |
| 20-25 | 趋势萌芽 | 准备入场 |
| 25-50 | 明确趋势 | 趋势跟随策略 |
| 50-75 | 极强趋势 | 持仓，但警惕反转 |
| 75-100 | 罕见极端趋势 | 立即获利了结 |

#### 加密货币应用

加密货币适合用 ADX(14) 在 4H 或日线判断 BTC/ETH 是否进入趋势市。当 ADX>25 且 +DI>-DI 时，配合 MACD 金叉可大幅提高入场胜率。ADX 不会告诉方向，仅告诉"是否有必要使用趋势策略"（来源：[Wikipedia – ADX](https://en.wikipedia.org/wiki/Average_directional_movement_index)）。

### 1.6 一目均衡表 (Ichimoku Kinko Hyo)

#### 计算原理

由日本记者细田悟一于二战前发明，是一套"一目了然"的综合性指标。

```python
# 转换线 (Tenkan-sen) - 9 周期中价均值
Tenkan = (Highest(High, 9) + Lowest(Low, 9)) / 2

# 基准线 (Kijun-sen) - 26 周期中价均值
Kijun = (Highest(High, 26) + Lowest(Low, 26)) / 2

# 先行带 A (Senkou Span A) - (转换线+基准线)/2，向前移 26 周期
SpanA = (Tenkan + Kijun) / 2
SenkouA = SpanA.shift(26)

# 先行带 B (Senkou Span B) - 52 周期中价均值，向前移 26 周期
SenkouB = ((Highest(High, 52) + Lowest(Low, 52)) / 2).shift(26)

# 延迟线 (Chikou Span) - 收盘价向后移 26 周期
Chikou = close.shift(-26)
```

#### 五线体系解读

| 线 | 别名 | 周期 | 作用 |
|----|------|------|------|
| 转换线 | Tenkan | 9 | 短期均衡价 |
| 基准线 | Kijun | 26 | 中期均衡价 |
| 先行带 A | Senkou A | (9+26)/2 | 未来 26 期的支撑/阻力 |
| 先行带 B | Senkou B | 52 | 更长期的支撑/阻力 |
| 延迟线 | Chikou | 26 | 验证趋势强度 |
| 云的厚度 | Kumo | - | 反映支撑/阻力强度 |

#### 核心信号

- **多头**：价格在云上，Tenkan>Kijun，Chikou>价格（Kumo 上方）
- **空头**：价格在云下，Tenkan<Kijun，Chikou<价格（Kumo 下方）
- **强信号**：Tenkan/Kijun 交叉 + 突破云
- **支撑/阻力**：云的上沿（Senkou A）和下沿（Senkou B）

#### 局限

- 元素过多，参数固定 (9,26,52,26)，不灵活
- 在加密货币这种 7×24 高波动市场，52 周期可能偏长
- 学习曲线陡峭，"云"经常被错误解读

#### 加密货币应用

BTC 日线 Ichimoku 在 2020-2021 牛市中，云层从阻力转为强支撑。一目均衡表在亚洲交易者中尤其流行，常与 K线形态结合使用（来源：[Investopedia – Ichimoku Cloud](https://www.investopedia.com/terms/i/ichimoku-cloud.asp)）。

## 2. 震荡指标 (Oscillators)

震荡指标在价格区间内摆动（通常 0-100 或 -100 到 0），用于识别**超买/超卖状态**及潜在的反转点。在盘整行情中表现优异，在强趋势中容易过早发出反转信号。

### 2.1 相对强弱指标 (RSI)

#### 计算原理

由 J. Welles Wilder 于 1978 年提出，衡量近期价格上涨与下跌幅度的比值（来源：[Wikipedia – RSI](https://en.wikipedia.org/wiki/Relative_strength_index)）。

```python
# 步骤 1：计算每期价格变动
delta = close - close.shift(1)
gain = max(delta, 0)
loss = max(-delta, 0)

# 步骤 2：使用 Wilder 平滑（等价于 alpha=1/14 的 EMA）
avg_gain = gain.ewm(alpha=1/14, adjust=False).mean()
avg_loss = loss.ewm(alpha=1/14, adjust=False).mean()

# 步骤 3：RS 与 RSI
RS = avg_gain / avg_loss
RSI = 100 - (100 / (1 + RS))
```

简化形式：
```latex
\text{RSI} = 100 \cdot \frac{\text{SMMA}(U, n)}{\text{SMMA}(U, n) + \text{SMMA}(D, n)}
```

#### 关键阈值与解读

| RSI 区间 | 含义 | 操作 |
|---------|------|------|
| > 70 | 超买（Overbought） | 警惕回调 |
| 50-70 | 多头占优 | 持有多单 |
| 30-50 | 空头占优 | 持有空单 |
| < 30 | 超卖（Oversold） | 警惕反弹 |
| 80-100 或 0-20 | 极端超买/超卖 | 反转概率高 |

#### 高级用法

1. **背离 (Divergence)**：
   - **看涨背离**：价格创新低，RSI 未创新低 → 强烈的看涨反转信号
   - **看跌背离**：价格创新高，RSI 未创新高 → 强烈的看跌反转信号
2. **失败摆动 (Failure Swing)**：RSI 突破 70 后回落再破前低形成双顶，是 Wilder 提出的**最强反转信号**之一
3. **趋势区间切换**：Andrew Cardwell 发现上升趋势中 RSI 主要在 40-80 区间运行，下降趋势中在 20-60 区间

#### 局限

- 在强趋势中会出现长期超买/超卖钝化（BTC 单边上涨时 RSI 可连续多周 >70）
- 默认 14 周期较慢，短线交易者常用 6-9 周期
- Cutler's RSI 改用 SMA 计算，消除了"数据长度依赖性"，但与 Wilder 版本有细微差异

#### 加密货币应用

BTC 月线 RSI 在 2021 年 11 月触及 92 的历史极值，随后开启了长达 1 年的熊市。ETH 在 2022 年 11 月 FTX 崩盘后周线 RSI 跌至 24.7，是十年内最深的超卖。加密货币市场建议**多周期共振**：周线 RSI<30 + 日线 RSI 底背离 = 强买入信号。

### 2.2 KDJ 指标

#### 计算原理

KDJ 是中国技术分析中广泛使用的随机指标衍生，由 K、D、J 三条曲线组成。原始参数为 (9, 3, 3)：

```python
# 步骤 1：未成熟随机值 RSV（9 日）
RSV = (close - Lowest(9)) / (Highest(9) - Lowest(9)) * 100

# 步骤 2：K 值（RSV 的 3 日 SMA 移动平均）
K = SMA(RSV, 3)  # 实际中是递归
# K_today = (RSV + 2 * K_yesterday) / 3

# 步骤 3：D 值（K 的 3 日 SMA）
D = SMA(K, 3)
# D_today = (K + 2 * D_yesterday) / 3

# 步骤 4：J 值（领先指标）
J = 3 * K - 2 * D
```

#### 三线含义

| 线 | 含义 | 特性 |
|----|------|------|
| K | 快速确认线 | 反映短期动能，对价格最敏感 |
| D | 慢速确认线 | K 的平滑，常作为信号线 |
| J | 超前/领先线 | 波动最大，K 的 3 倍偏离度 |

J 值可以 > 100 或 < 0，这是 KDJ 与 RSI/KDJ 不同之处。

#### 应用场景

- **金叉**：K 上穿 D 且 K<20，强烈买入信号
- **死叉**：K 下穿 D 且 K>80，强烈卖出信号
- **钝化问题**：BTC 在单边行情中 KDJ 长时间位于 80-100 区间，传统超买信号失效

#### 局限

- **严重钝化**：在加密货币单边行情中，KDJ 可连续数日维持 90+，失去超买意义
- 期货/合约市场中 KDJ 钝化尤其严重
- 中国市场的传统指标，西方交易者使用较少

#### 加密货币应用

中国加密货币交易员最常使用 KDJ。**改良用法**：配合 30 分钟级别成交量 + EMA(21) 方向使用，避免钝化陷阱。BTC 4H KDJ 金叉 + 日线 MACD 零轴上方 = 较为可靠的短线买点。

### 2.3 随机指标 (Stochastic Oscillator)

#### 计算原理

由 George Lane 在 1950 年代末期发明，假设上涨趋势中收盘价倾向于接近周期高点，下跌趋势中接近低点（来源：[Wikipedia – Stochastic Oscillator](https://en.wikipedia.org/wiki/Stochastic_oscillator)）。

```python
# 慢速随机 %K
%K = 100 * (close - Lowest(14)) / (Highest(14) - Lowest(14))

# 慢速随机 %D (%K 的 3 期 SMA)
%D = SMA(%K, 3)

# 慢速 %D (%D 的 3 期 SMA)，用于 3 线 Stochastic
%D_Slow = SMA(%D, 3)
```

#### 信号类型

1. **金叉/死叉**：%K 上穿/下穿 %D
2. **超买超卖**：> 80 为超买，< 20 为超卖
3. **背离**：%K 与价格走势的顶/底背离

#### 慢速 vs 快速随机

- **快速随机 (Fast Stochastic)**：%K 由原始 RSV 计算，%D 是 %K 的 3 期 SMA → 信号多但杂音大
- **慢速随机 (Slow Stochastic)**：%K 是 %D_Fast，%D 是 %K 的 3 期 SMA → 信号平滑，更常用

#### 局限

- 在强趋势中发出大量错误反转信号
- 对突发事件反应慢（依赖 N 期高低点）
- 与 RSI、Williams %R 本质上同源

#### 加密货币应用

常用于 1H、4H 周期短线反转。StochRSI 是 Stoch + RSI 的合成（先对 RSI 计算 Stoch），对加密货币更敏感。

### 2.4 威廉指标 (Williams %R)

#### 计算原理

由 Larry Williams 于 1979 年提出，与 %K Stochastic 在数学上完全等价，但映射到 -100 到 0 区间（来源：[Wikipedia – Williams %R](https://en.wikipedia.org/wiki/Williams_%25R)）。

```latex
\%R = \frac{\text{Highest High}_N - \text{Close}}{\text{Highest High}_N - \text{Lowest Low}_N} \times (-100)
```

#### 阈值

| %R 范围 | 含义 |
|---------|------|
| 0 到 -20 | 超买 |
| -80 到 -100 | 超卖 |

#### Williams 交易规则

- 买入：%R 达到 -100 → 5 天后 → %R 上升到 -95 以上
- 卖出：%R 达到 0 → 5 天后 → %R 下降到 -5 以下

#### 加密货币应用

%R 对加密货币 1H 级别超买超卖反应灵敏，常与 MACD 配合。默认周期 14 适合日线，4H 建议用 9-12。

### 2.5 顺势指标 (CCI)

#### 计算原理

由 Donald Lambert 于 1980 年提出，通过比较当前价格与统计平均的偏离来识别周期性反转。

```python
# 典型价格 (TP)
TP = (High + Low + Close) / 3

# 移动平均
SMA_TP = SMA(TP, 20)

# 平均绝对偏差
MAD = SMA(abs(TP - SMA_TP), 20)

# CCI
CCI = (TP - SMA_TP) / (0.015 * MAD)
```

#### 阈值

- 常规阈值：+100 / -100
- **极限阈值**：+200 / -200（更严格的反转信号，但信号稀少）

#### 局限

- 在强趋势中 CCI 可远超 ±200，传统超买信号失效
- 0.015 常数是历史经验值，导致大多数读数落在 ±100 内

#### 加密货币应用

BTC 4H 周期 CCI 突破 +200 是强烈超买警告，常对应阶段顶部。配合成交量使用效果更佳。

### 2.6 动量指标 (MOM) 与变化率 (ROC)

#### 计算原理

```python
# 动量 (Momentum)
MOM = close - close.shift(N)

# 变化率 (Rate of Change)
ROC = (close / close.shift(N) - 1) * 100
```

MOM 是绝对值，ROC 是百分比，二者本质相同。

#### 应用

- 衡量价格变化的速度和幅度
- ROC 穿越零线作为趋势变化信号
- 与移动平均配合形成"双指标"系统

#### 局限

- 对价格尖刺极度敏感
- 在加密货币中常出现"假突破"

#### 加密货币应用

MOM(10) 用于识别 BTC 短期动能变化。ROC(20) 常用于山寨币相对强度排序（与 BTC 比较的强弱）。

## 3. 布林带 (Bollinger Bands)

由 John Bollinger 在 1980 年代发明，是**结合趋势 + 波动率**的复合指标（来源：[Wikipedia – Bollinger Bands](https://en.wikipedia.org/wiki/Bollinger_Bands)）。

### 3.1 计算原理

```python
# 中轨 (Middle Band) - 20 日 SMA
Middle = SMA(close, 20)

# 标准差
sigma = close.rolling(20).std(ddof=0)  # 总体标准差，除以 n

# 上轨 / 下轨
Upper = Middle + 2 * sigma
Lower = Middle - 2 * sigma
```

**默认参数 (20, 2)**：20 日 SMA + 2 倍标准差。Bollinger 强调使用**总体标准差 (除以 n)** 而非样本标准差 (除以 n-1)。

### 3.2 衍生指标

#### 带宽 (Bandwidth)
```python
Bandwidth = (Upper - Lower) / Middle
```
反映**相对波动率**。带宽上升 → 波动扩大；带宽收缩 → 波动收敛。

#### %b 指标
```python
%b = (close - Lower) / (Upper - Lower)
```
- %b = 0：价格在下轨
- %b = 1：价格在上轨
- %b > 1 或 < 0：价格突破布林带

### 3.3 收口/开口 (Squeeze) 解读

| 状态 | 含义 | 应对 |
|------|------|------|
| **Squeeze (收口)** | 带宽处于历史 10% 分位以下 → 波动率极低 | 准备大行情突破 |
| **Breakout (开口)** | 带宽从低位快速扩张 | 趋势初期，可能追涨杀跌 |
| **Walking the Bands** | 价格沿上轨/下轨运行 | 强趋势，不要逆势 |
| **M-Top / W-Bottom** | 形态反转信号 | 配合 RSI 确认 |

### 3.4 与 RSI 配合

经典组合：**"双布林带 + RSI" 策略**
- 收盘价触及下轨 + RSI < 30 → 强买入
- 收盘价触及上轨 + RSI > 70 → 强卖出
- **背离信号**：价格触及上轨新高 + RSI 顶背离 = 强反转

### 3.5 局限

- 学术研究（Lento 2007, Balsara 2007）显示传统布林带突破策略**在 1995 年后无明显超额收益**，反向策略（"逆布林"）反而表现更好。
- 实证显示价格**仅 88% 时间位于带内**（而非正态分布假设的 95%），因为价格分布具有**肥尾 (fat tails)** 特征。
- 在加密货币高波动市场，2 倍标准差可能过窄，建议调整为 2.5-3 倍。

### 3.6 加密货币应用

BTC 4H 布林带 Squeeze → 突破常对应 5-10% 单日行情。%b 突破 1.0 + RSI>80 = 顶部预警。带宽与 ATR 结合可作为仓位调整依据。

## 4. 成交量指标

**Price × Volume = Money Flow**。价格变动需要成交量确认，缺乏成交量的"假突破"通常很快被反噬。

### 4.1 能量潮 (OBV)

由 Joseph Granville 于 1963 年提出，是最简单的累计成交量指标（来源：[Wikipedia – OBV](https://en.wikipedia.org/wiki/On-balance_volume)）。

```python
# 简化伪代码
if close > close_prev:
    OBV_today = OBV_yesterday + volume
elif close < close_prev:
    OBV_today = OBV_yesterday - volume
else:
    OBV_today = OBV_yesterday
```

#### 核心用法

- **趋势确认**：价格、OBV 同向上涨/下跌 → 健康趋势
- **背离**：
  - 看跌背离：价格创新高，OBV 未创新高 → 上涨乏力
  - 看涨背离：价格创新低，OBV 未创新低 → 抛压减弱
- **突破验证**：放量突破关键阻力位 + OBV 同向突破 = 真突破

#### 局限

- 是**累计值**，起算点任意，绝对数值无意义
- 对单一价格水平过于敏感（仅看 close）

#### 加密货币应用

BTC 2021 年 4 月 6.4 万美元顶部时出现**明显 OBV 顶背离**，随后暴跌 50%。SOL、DOGE 等山寨币 OBV 走势与 BTC 联动性强，可作为领先指标。

### 4.2 成交量加权平均价 (VWAP)

#### 计算原理

```python
# 典型价
typical_price = (High + Low + Close) / 3
# 当日累计
cumvol_price = (typical_price * volume).cumsum()
cumvol = volume.cumsum()
# VWAP
VWAP = cumvol_price / cumvol
```

通常**从 UTC 00:00 开始**逐根 1 分钟/5 分钟 K 线重置计算。

#### 应用场景

- **机构交易基准**：大资金常以 VWAP 算法拆单减少市场冲击
- **日内方向**：价格在 VWAP 之上为偏多，之下为偏空
- **支撑阻力**：VWAP 本身常作为日内动态支撑/阻力
- **机构痕迹**：价格反复测试 VWAP 后远离 = 机构强势介入

#### 局限

- 严格定义是**日内**指标，隔夜跳空后失去意义
- 多用于股票/期货，加密货币 7×24 场景下需要选择"交易日"起点

#### 加密货币应用

BTC/ETH 现货大单常以 VWAP 为基准。在 1 分钟/5 分钟 K 线观察 VWAP 与价格关系是短线高手的常用技术。**Anchor VWAP** (aVWAP) 从特定历史事件（如 ETF 批准、FOMC 会议）开始累计，被专业交易者广泛使用。

### 4.3 资金流量指标 (MFI)

RSI 的"音量版"，结合价格与成交量（来源：[Wikipedia – MFI](https://en.wikipedia.org/wiki/Money_flow_index)）。

```python
# 典型价
TP = (High + Low + Close) / 3
# 资金流量
MF = TP * Volume

# 14 日累计
positive_MF = MF[TP > TP.shift(1)].rolling(14).sum()
negative_MF = MF[TP < TP.shift(1)].rolling(14).sum()

# MFI
MR = positive_MF / negative_MF
MFI = 100 - 100 / (1 + MR)
```

#### 阈值与解读

| MFI | 含义 |
|-----|------|
| > 80 | 强超买，资金疯狂流入 |
| 20-80 | 中性区间 |
| < 20 | 强超卖，资金大量流出 |

#### 与 RSI 的差异

| 维度 | RSI | MFI |
|------|-----|-----|
| 输入 | 收盘价 | 典型价 × 成交量 |
| 反映 | 价格动量 | 资金流强度 |
| 假信号 | 多 | 较少（多了成交量验证） |

#### 加密货币应用

MFI 在山寨币中比 RSI 可靠，因为它能识别"无量上涨"。Marek & Čadková (2020) 研究显示 MFI 经参数优化后**可显著跑赢买入持有策略**。

### 4.4 累积/派发线 (A/D Line)

由 Marc Chaikin 提出，与 OBV 类似但用**收盘在当日波幅中的位置**加权（来源：[Wikipedia – A/D Index](https://en.wikipedia.org/wiki/Accumulation/distribution_index)）。

```python
# CLV (Close Location Value) 范围 [-1, +1]
CLV = ((close - low) - (high - close)) / (high - low)
# A/D Line
A/D = A/D.shift(1) + volume * CLV
```

#### 解读

- **A/D 上升 + 价格上升**：强健康趋势（量价齐升）
- **A/D 下降 + 价格上升**：看跌背离，可能是派发阶段
- **A/D 上升 + 价格下降**：看涨背离，可能是吸筹阶段

#### 与 OBV 的比较

| 维度 | OBV | A/D Line |
|------|-----|----------|
| 权重依据 | 收盘涨跌 (二值) | 收盘在波幅中位置 (连续值) |
| 灵敏度 | 较低 | 较高 |
| 适用 | 长期趋势 | 中期反转 |

#### 加密货币应用

A/D 在 BTC 4H/日线对大型洗盘/吸筹识别力强。

### 4.5 蔡金资金流量 (CMF)

Chaikin Money Flow 是 A/D Line 的**短期版本**（20 期窗口）。

```python
MFM = ((close - low) - (high - close)) / (high - low)
MFV = MFM * volume
CMF = MFV.rolling(20).sum() / volume.rolling(20).sum()
```

| CMF | 含义 |
|-----|------|
| > 0.05 | 资金净流入 |
| -0.05 到 0.05 | 中性 |
| < -0.05 | 资金净流出 |

CMF 比 A/D Line 更适合**短线周期**（5 日、20 日），在加密货币中常用于识别 1H-4H 资金动向。

### 4.6 成交量分布图 (Volume Profile)

与传统的时间-价格图表不同，Volume Profile 在**价格轴**上显示每个价格区间的成交量。

#### 关键概念

| 名称 | 定义 | 作用 |
|------|------|------|
| **POC (Point of Control)** | 成交量最大的单一价格 | 强磁吸价位 |
| **VAH (Value Area High)** | 70% 成交量的上边界 | 阻力位 |
| **VAL (Value Area Low)** | 70% 成交量的下边界 | 支撑位 |
| **HVN (High Volume Node)** | 成交量高点 | 支撑/阻力 |
| **LVN (Low Volume Node)** | 成交量低点 | 价格快速穿越 |

#### 应用

- 识别**机构建仓价格区**（HVN 通常是吸筹区）
- 判断价格是"价值修复"（向 VAH/VAL 回归）还是"趋势延伸"
- 与水平支撑阻力结合效果更佳

#### 加密货币应用

Glassnode、Coinglass 提供 BTC/ETH 的 Volume Profile 数据。POC 常成为后续行情的中枢价，VAH/VAL 是设置止损止盈的关键参考。

## 5. 波动率指标

波动率指标衡量价格变动剧烈程度，是**仓位管理和止损设置**的关键工具。

### 5.1 平均真实波幅 (ATR)

由 Wilder 在 1978 年提出。

```python
# 真实波幅 (True Range)
TR = max(
    High - Low,                    # 当日波幅
    abs(High - close_prev),        # 跳空高开
    abs(Low - close_prev)          # 跳空低开
)
# ATR (14 日 Wilder 平滑)
ATR = TR.ewm(alpha=1/14, adjust=False).mean()
```

#### 仓位管理应用

```python
# 方法 1：固定 ATR 止损
stop_loss = entry_price - 2 * ATR

# 方法 2：基于账户风险
position_size = account_risk / (entry_price - stop_loss)
# 其中 stop_loss = entry - k * ATR (k 常取 1.5-3)
```

#### 加密货币应用

BTC 日线 ATR 在 2021 年 5 月 19 日"519 崩盘"前后从 2000 飙升至 6000 美元。**波动率上升 → 减小杠杆**是核心法则。建议持仓风险 = 账户的 1-2%，止损 = 1.5-2 倍 ATR。

### 5.2 历史波动率 (HV)

```python
# N 日对数收益率标准差
log_return = np.log(close / close.shift(1))
HV_N = log_return.rolling(N).std() * sqrt(252)  # 年化
```

HV(30) 反映过去 1 个月实际波动。BTC 历史波动率年化通常在 50-100%，远高于股票（S&P 500 约 15%）。

### 5.3 隐含波动率 (IV)

来自期权定价模型（B-S）的"市场预期波动率"。

```python
# Black-Scholes 反解
# C = S*N(d1) - K*exp(-rT)*N(d2)
# 输入市场价格，反解 sigma 即为 IV
```

Deribit 是加密货币期权最大交易所，提供 BTC/ETH 期权 IV 数据。

| 指标 | 含义 | 用途 |
|------|------|------|
| **IV 本身** | 市场对未来波动的预期 | 期权定价 |
| **DVOL** | Deribit 波动率指数 | 类似 VIX，反映恐慌 |
| **IV/HV 比** | 隐含/历史波动率 | 期权是否昂贵 |
| **IV 期限结构** | 不同到期日的 IV 差异 | 事件预期 |

DVOL 在 2020 年 3 月 12 日达到 166 的历史峰值。

### 5.4 唐奇安通道 (Donchian Channels)

由 Richard Donchian 发明，20 世纪最经典的趋势跟踪指标之一。

```python
# 上轨 = N 日最高价
Upper = Highest(High, N)
# 下轨 = N 日最低价
Lower = Lowest(Low, N)
# 中轨 = (上轨 + 下轨) / 2
Middle = (Upper + Lower) / 2
```

#### 海龟交易法则

经典 N=20（突破）入场，N=10（止损）出场。

#### 加密货币应用

在 BTC/ETH 日线 N=20 通道突破是经典趋势策略。缺点是**反转动能差**，需配合 ATR 止损。

## 6. 加密货币专属指标

传统指标主要基于价格和成交量，但加密货币市场有独特的链上数据与衍生品数据。

### 6.1 资金费率 (Funding Rate)

永续合约（Perpetual Futures）特有的多空平衡机制。

```python
# Funding Rate 由交易所计算（通常每 8 小时一次）
# 公式（简化）：
funding_rate = clamp(
    (mark_price_avg - index_price_avg) / index_price_avg / funding_interval,
    -0.75%, 0.75%  # Binance 上限
)
```

#### 解读

| Funding Rate | 含义 | 交易含义 |
|--------------|------|---------|
| > 0.03% | 多头支付给空头 | 市场过热，多头拥挤 |
| 接近 0 | 多空平衡 | 方向不明确 |
| < -0.03% | 空头支付给多头 | 市场过冷，空头拥挤 |
| > 0.1% | 极端拥挤 | 警惕"插针"反转 |

#### 加密货币应用

资金费率是**情绪温度计**。BTC 在 2021 年 4 月、2021 年 11 月顶部时，资金费率均超过 0.1%。结合 OI（持仓量）使用：
- **价格上涨 + 资金费率上升 + OI 上升** = 健康上涨
- **价格上涨 + 资金费率负 + OI 下降** = 空头平仓，趋势脆弱
- **Funding Rate 反转** = 短期趋势可能反转

数据源：Binance、Bybit、Coinglass、Coinalyze。

### 6.2 多空持仓比 (Long/Short Ratio)

```python
# 大户持仓比 = 持有大额头寸账户的多空比
# 全市场账户数比 = 所有账户多空比
long_short_ratio = long_accounts / short_accounts
```

| 比值 | 含义 |
|------|------|
| > 1.5 | 散户看多（看跌信号） |
| 0.7-1.5 | 平衡 |
| < 0.7 | 散户看空（看涨信号） |

**反指逻辑**：当散户极度看多时，反而可能是顶部；与"聪明钱"指标（大户持仓比）结合使用效果更佳。

### 6.3 爆仓数据 (Liquidations)

加密货币高杠杆市场独有的现象。

```python
# 爆仓金额（USD）按周期累计
long_liquidations = sum(notional_value where margin_call == True and position == 'long')
short_liquidations = sum(notional_value where margin_call == True and position == 'short')
```

#### 应用

- **多空爆仓比**：24h 多单爆仓 / 空单爆仓 → 短期情绪
- **大额爆仓**（>$10M）：常引发连锁平仓，推动趋势加速
- **爆仓后修复**：大量爆仓后 24-48h 容易出现技术性反弹
- **反向指标**：爆仓高峰后，市场往往已"出清"风险

2021 年 5 月 19 日全网 24h 爆仓金额超 80 亿美元，标志 BTC 阶段性底部。

### 6.4 主动买卖比 (Taker Buy/Sell Ratio)

```python
taker_buy_volume = sum(volume where taker == 'buy')
taker_sell_volume = sum(volume where taker == 'sell')
taker_buy_sell_ratio = taker_buy_volume / taker_sell_volume
```

| 比值 | 含义 |
|------|------|
| > 1.2 | 主动买盘强 |
| 0.8-1.2 | 平衡 |
| < 0.8 | 主动卖盘强 |

**Taker** = 在订单簿上主动成交的一方（吃单方），反映真实的方向意愿。

#### 与资金费率的配合

- 价格上涨 + 资金费率正 + Taker Buy/Sell > 1.2 = **强多头趋势**
- 价格上涨 + 资金费率负 + Taker Buy/Sell > 1.2 = **空头投降，反弹接近尾声**

### 6.5 交易所净流量 (Exchange Netflow)

```python
exchange_netflow = inflow - outflow
# 正数：净流入交易所（潜在抛压）
# 负数：净流出交易所（潜在囤币）
```

#### 解读

| 状态 | 含义 |
|------|------|
| **大额净流入** | 投资者准备卖出（看跌信号） |
| **大额净流出** | 投资者提至冷钱包（看涨信号） |
| **持续净流出 + 价格下跌** | 强信号 - 持有者拒绝抛售 |

数据源：Glassnode、CryptoQuant、IntoTheBlock。2022 年 11 月 FTX 崩盘后，BTC 交易所余额持续下降，是 2023 年反弹的领先指标。

### 6.6 稳定币市值变化 (Stablecoin Supply)

```python
# 稳定币总市值 = USDT + USDC + DAI + TUSD + ...
# 稳定币在交易所的余额 = 潜在购买力
```

| 指标 | 含义 |
|------|------|
| **稳定币总市值上升** | 新资金入场（看涨） |
| **交易所稳定币余额上升** | 准备买入（强烈看涨） |
| **USDC 市值快速下降** | 美元赎回/恐慌（看跌） |
| **USDT 溢价（CNY/USD）** | 亚洲市场情绪 |

USDC 在 2022 年 3 月 SVB 事件中市值一周缩水 100 亿美元，伴随 BTC 大跌。

## 7. 指标组合策略

### 7.1 趋势 + 震荡组合

**核心逻辑**：用趋势指标定方向，用震荡指标找入场点。

#### 经典组合 1：MA + RSI

```python
# 买入条件
trend_up = close > MA(200)  # 长期趋势向上
entry = (RSI(14) < 35) & (close > MA(200))  # 长期上升 + 短期超卖
```

#### 经典组合 2：MACD + 布林带

```python
# 买入条件
macd_bull = DIF > DEA and DIF < 0  # MACD 金叉且在零轴下（多头动能初起）
bb_oversold = close < Lower_BB(20, 2)  # 价格触及下轨
entry = macd_bull and bb_oversold
```

#### 经典组合 3：ADX + SAR

```python
# 趋势确认入场
strong_trend = ADX(14) > 25
sar_bullish = close > SAR(0.02, 0.20)
entry = strong_trend and sar_bullish
```

### 7.2 多指标共振 (RSI + MACD + KDJ 三重共振)

**共振思想**：多个指标同时发出相同信号时，信号强度最高。

```python
# 三重共振买入
rsi_bull = (RSI(14) > 50) & (RSI(14) < 70)  # 中性偏多
macd_bull = (DIF > DEA) & (HIST > 0)          # 金叉 + 柱状图为正
kdj_bull = (K > D) & (K < 80) & (J < 100)     # 金叉 + 未超买

buy_signal = rsi_bull & macd_bull & kdj_bull
```

**优势**：三重共振的胜率通常比单一指标提高 10-20%。

**代价**：信号稀少，可能错过大行情。

#### 实操建议

- **多时间框架 (MTF) 共振**：周线趋势 + 日线动量 + 4H 入场点
- **多类型共振**：趋势指标 + 震荡指标 + 成交量指标 + 链上指标

### 7.3 指标的滞后性与"指标陷阱"

#### 滞后性的本质

所有基于历史价格的指标都是**滞后指标 (Lagging Indicator)**：
- 价格先动 → 指标跟随 → 信号产生
- 在趋势转折点，指标会给出错误信号

#### 五大常见陷阱

1. **假突破陷阱**：价格突破 MA/布林带后快速回归 → 趋势指标假信号
2. **钝化陷阱**：BTC 单边行情中 RSI/KDJ 持续超买/超卖，信号失效
3. **背离陷阱**：指标背离后价格继续沿原方向运行（Wilder 警告：背离在趋势中可能持续多周期）
4. **周期错配陷阱**：在 4H 用日线参数计算 RSI，灵敏度严重失配
5. **流动性陷阱**：低流动性山寨币中，1% 资金即可推动 50% 行情，技术指标完全失效

#### 应对原则

- 永远在**大时间框架**确认趋势（周线 > 日线 > 4H > 1H > 15M）
- **不要在盘整行情使用趋势指标**
- **不要在强趋势中使用震荡指标做反转**
- 单一指标出现信号后，等待**多指标共振**再行动
- 任何指标信号都必须配合**风险控制（止损/仓位）**

### 7.4 指标参数优化陷阱 (Over-fitting)

#### 核心问题

通过对历史数据的反复回测"调优"参数，得到的"最佳参数"在**样本外**往往完全失效。

#### 学术研究结论

- **Marek & Šedivá (2017)**：RSI 经典参数 (14) 在测试期内能产生超额收益，但**长期被买入持有策略超越**。
- **Marek & Čadková (2020)**：MFI 推荐的固定参数**没有优势**，必须对每只标的单独优化才有效——但这本身加剧了过拟合。
- **Lento et al. (2007)**：布林带标准参数在 1995 年后无明显超额收益。
- **Butler et al. (2012)**：通过粒子群算法优化布林带参数后，**样本外表现**得到改善，但代价是**策略复杂度和维护成本**显著上升。

#### 过拟合的典型表现

| 现象 | 原因 |
|------|------|
| 回测年化 500%+ | 过度拟合或前视偏差 |
| 夏普比率 > 3 | 数据窥探 (Data Snooping) |
| 最大回撤极小 | 未考虑真实滑点和流动性 |
| 胜率 90%+ | 高度参数优化，未考虑样本外 |

#### 应对原则

1. **样本外测试 (Out-of-Sample Testing)**：将数据分为训练集和测试集
2. **跨标的/跨周期验证**：同一参数在 BTC、ETH、SOL 上是否都有效
3. **Walk-Forward Analysis**：滚动窗口优化
4. **简单参数优先**：优先使用经典参数 (RSI=14, MACD=12/26/9, BB=20/2)
5. **避免参数过细**：参数越少越稳健
6. **考虑交易成本**：0.1% 手续费对短线策略的杀伤极大
7. **流动性约束**：回测时假设滑点 0.05-0.1%，并限制最大仓位占比

#### 推荐参数原则

| 指标 | 经典参数 | 调整建议 |
|------|---------|---------|
| MA | 20, 50, 200 | 加权考虑斐波那契数 (13, 21, 55) |
| EMA | 9, 21, 55 | 与 MA 共用 |
| MACD | 12, 26, 9 | 短线 5, 35, 5 |
| RSI | 14 | 短线 6-9，长线 21-25 |
| KDJ | 9, 3, 3 | 加密货币建议 14, 3, 3 减少钝化 |
| Stochastic | 14, 3, 3 | 短线 9, 3, 3 |
| 布林带 | 20, 2 | 高波动市场用 2.5-3 |
| ATR | 14 | 周期不变 |
| OBV | N/A | 建议 EMA 平滑 (20) |
| MFI | 14 | 短线 9 |

## 结语

技术指标是加密货币交易者的"工具箱"，但**没有银弹**。本报告系统梳理了 7 大类、25+ 个常用指标，涵盖趋势、震荡、波动、成交量、链上与衍生品数据。然而，决定成败的往往是：

1. **多时间框架分析 (MTF)** — 永远从大周期看趋势
2. **多类型指标共振** — 趋势+震荡+成交量+链上
3. **严格的风险管理** — 仓位控制 + 止损纪律
4. **对市场结构的理解** — 庄家行为、流动性周期、宏观环境
5. **避免过度优化** — 简单稳健的参数优于高度调优

> **量化先贤名言**："市场可以被技术分析者观察，但不能被技术分析者预测。" (J. Welles Wilder, 1978)
