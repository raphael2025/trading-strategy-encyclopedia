## 一、高频交易（HFT）基础

### 1. 定义与特征

**HFT（High-Frequency Trading）**：
- 毫秒至秒级的交易
- 极快速度执行
- 大量订单
- 持仓时间极短

**关键特征**：
- **速度**：低延迟基础设施
- **频率**：每秒数千订单
- **数据**：Level 2/3 行情
- **盈利**：单笔极小（$0.0001-0.01），但量大

### 2. HFT 分类

**做市 HFT**（50-60% 交易量）：
- 在订单簿两侧挂单
- 赚买卖价差
- 主要收入来源

**套利 HFT**（10-15%）：
- 跨交易所、跨资产
- 三角套利
- 闪电贷 + DEX

**订单流 HFT**（20-30%）：
- 抢先交易（但合规要求变化）
- 订单簿分析

**新闻/事件 HFT**（5-10%）：
- 自动化新闻分析
- 快速对宏观事件反应

### 3. 加密 HFT 与传统差异

| 维度 | 传统 HFT | 加密 HFT |
|------|----------|----------|
| 撮合速度 | 纳秒级（交易所直连） | 毫秒级（API 限制） |
| 交易时间 | 6.5-23h/日 | 24/7 |
| 延迟 | 共置服务器 | API 限速 |
| 监管 | SEC/CFTC 严管 | 不一致 |
| 准入门槛 | 数千万美元 | 10-100 万美元即可 |
| 主要策略 | 做市+抢跑+统计套利 | 做市+跨所+三角+DeFi 套利 |

### 4. 加密 HFT 案例

**Alameda Research**（已倒闭）：
- SBF 2017 年创立
- 2021 年峰值日交易量 50-100 亿美元
- 2022-11 暴雷
- 教训：单点风险、对手方风险、监管风险

**Wintermute**（现役领先）：
- 2017 年 Evgeny Gaevoy 创立
- 2024 年日交易量 50-100 亿美元
- 2022 年 9 月被黑客攻击 1.6 亿美元（已恢复）
- 现货 + 衍生品 + DeFi

**Jump Crypto**：
- 母公司 Jump Trading 1999 年创立
- 加密部门 2018 年成立
- Solana、 Wormhole 重要支持方
- 2022 年 Wormhole 被黑 3.2 亿美元（Jump 注资救回）

**Cumberland（DRW）**：
- DRW 1992 年创立
- Cumberland 是加密部门
- 场外交易 + 做市
- 大宗交易专家

**Flow Traders**：
- 阿姆斯特丹上市

**Tower Research**：

## 二、做市商模型

### 1. 做市基础

**做市商**：
- 在订单簿**两侧挂限价单**
- 买入价（Bid）< 卖出价（Ask）
- 价差（Spread）= Ask - Bid
- 价差就是做市商的"毛利"

**流程**：
1. 评估库存（已持有的资产）
2. 调整两侧价格
3. 持续提供流动性
4. 赚取价差

### 2. 库存风险

**核心问题**：
- 做市商持有多头头寸
- 价格上涨 → 库存价值↑（机会成本）
- 价格上涨 → 卖出后拿钱 → 错过继续上涨

**应对**：
- 动态调整价格（价格上涨时减少买单）
- 多资产对冲
- 风险限额

### 3. 经典模型

**Ho-Stoll 模型**（1981）：
- 第一个学术模型
- 库存作为状态变量
- 价格调整基于库存

**Avellaneda-Stoikov 模型**（2008）：
- 实用性最强
- 包含库存风险厌恶
- 公式化价差

#### Avellaneda-Stoikov 详解

**核心思想**：
- 报价 = 平衡（赚更多 vs 减少库存风险）
- 公式：

**中间价（reservation price）**：
- r(s, q, t) = s - q × γ × σ² × (T - t)
- s = 标的价格
- q = 库存（正=多头，负=空头）
- γ = 风险厌恶系数
- σ = 波动率
- T-t = 距平仓时间

**价差**：
- δ = γ × σ² × (T - t) + (2/γ) × ln(1 + γ/κ)
- κ = 订单到达率参数

**买价**：
- Bid = r - δ/2

**卖价**：
- Ask = r + δ/2

**实战**：
- γ ↑ → 价差宽（风险厌恶高）
- q ↑（多头多）→ 中间价下移（鼓励卖）
- σ ↑ → 价差宽

### 4. 实战应用

**Hummingbot 实现**：
- 开源做市框架
- 集成 Avellaneda 策略
- 2023 年处理 30+ 亿美元交易量
- 支持 30+ 交易所

**实战代码**（Hummingbot 风格）：
```python
# 简化版 Avellaneda 策略
def calculate_quotes(s, q, sigma, gamma, T_minus_t, kappa):
    # 中间价
    r = s - q * gamma * sigma**2 * T_minus_t
    
    # 价差
    delta = gamma * sigma**2 * T_minus_t + (2/gamma) * np.log(1 + gamma/kappa)
    
    # 两侧报价
    bid = r - delta / 2
    ask = r + delta / 2
    
    return bid, ask

# 使用
mid_price = 50000  # BTC
inventory = 0.5    # 0.5 BTC 多头
volatility = 0.02  # 日波动率 2%
gamma = 0.1
T_minus_t = 0.001  # 短持仓
kappa = 1.5

bid, ask = calculate_quotes(mid_price, inventory, volatility, gamma, T_minus_t, kappa)
print(f"Bid: {bid}, Ask: {ask}, Spread: {ask-bid}")
```

### 5. 其他做市模型

## 三、HFT 策略详解

### 1. 抢跑交易（Front Running）

**定义**：
- 检测到客户大单
- 抢在客户前下单
- 赚取滑点

**历史**：
- 2010 年代在传统市场猖獗
- 2014 年 IEX 推出"Speed Bump"对抗
- 2016 年 SEC 加强监管
- **加密市场**：相对宽松

**加密市场形式**：
- Memecoin 抢跑（sniper bots）
- DEX 抢先交易（sandwich attack）
- 大宗 OTC 抢跑

**争议**：
- 道德上模糊
- 法律上视情况
- MEV（最大可提取价值）成为新概念

### 2. MEV（Maximal Extractable Value）

**定义**：
- 区块链特有
- 验证者/搜索者通过重新排序交易获利
- 学术：Daian et al. 2020 "Flash Boys 2.0"

**主要形式**：
- **套利**：DEX 间价差
- **清算**：DeFi 协议清算
- **三明治攻击**：抢在用户交易前后下单
- **夹子交易（sandwich）**：大单前后夹击

**数据**（2024）：
- Ethereum MEV：年化 5-10 亿美元
- 主要通过 Flashbots 提取
- Solana MEV：年化 3-5 亿美元

**案例**：
- **jaredfromsubway.eth**：2022 年通过夹子攻击获利 700+ 万美元
- 多个 MEV bot 月入百万美元

### 3. 闪电贷套利

**定义**：
- Aave、dYdX 等协议的免抵押贷款
- 必须在同一交易内归还
- 用于套利

**例**：
1. 借 1000 万美元（无抵押）
2. 在 Uniswap 买入 ETH
3. 在 SushiSwap 卖出 ETH（更贵）
4. 归还借款 + 手续费
5. 利润 = 价差 - 手续费

**风险**：
- 套利空间已消失（被 MEV 抢走）
- 抢跑机器人竞争激烈
- 矿工/验证者合谋

### 4. 跨交易所套利

**流程**：
1. 同时监控多个交易所
2. 检测价差
3. 同步执行
4. 价差收敛，盈利

**关键**：
- 速度
- 资金预存
- API 限速

**典型价差**：
- BTC：< 0.05%（几乎无机会）
- 山寨币：0.1-1%
- 新币：可达 5%+

### 5. 跨链套利

**逻辑**：
- 同一币种在不同链
- 桥接延迟
- 流动性分散

**挑战**：
- 桥接时间长（5-30 分钟）
- 桥接费（0.1-1%）
- 跨链 MEV 风险

**应用**：
- ETH（Ethereum）vs ETH（Arbitrum）vs ETH（Optimism）
- USDC 多链流通
- 跨链桥套利

## 八、做市商风险

### 1. 库存风险

**案例**：
- 2021 年 5 月 19 日：BTC 跌 30%
- 做市商库存损失 5-15%
- 极端行情下风险极大

**应对**：
- 风险限额（单边头寸 ≤ 5%）
- 强制对冲
- 熔断机制

### 2. 对手方风险

**案例**：
- FTX 暴雷（2022）：Alameda 损失 80 亿美元
- 多个做市商在 FTX 存款

**应对**：
- 分散交易所
- 严格限额
- 储备证明验证

### 3. 模型风险

**案例**：
- 2020-03-12：波动率飙升
- Avellaneda 模型假设短期均值回归
- 极端行情下模型失效

**应对**：
- 动态参数
- 熔断机制
- 多模型

### 4. 技术风险

**案例**：
- API 故障
- 网络中断
- 软件 bug

**应对**：
- 多重冗余
- 监控告警
- Kill switch

### 5. 监管风险

**风险**：
- 操纵市场调查
- 监管政策变化
- 制裁

**应对**：
- 合规团队
- 多法域注册
- 与监管对话

## 九、实战：做市商启动

### 1. 资金需求

**初期**（100-500 万美元）：
- 单一交易所
- 单一币种
- 手动监控

**中期**（500 万-5000 万美元）：
- 多交易所
- 多币种
- 自动化

**高级**（5000 万+ 美元）：
- 多市场
- 多策略
- 全球基础设施

### 2. 团队需求

**最小团队**（2-3 人）：
- 1 量化研究员
- 1 工程师
- 1 交易员

**理想团队**（5-10 人）：
- 量化研究 2-3 人
- 工程 2-3 人
- 风控 1 人
- 业务 1-2 人

### 3. 时间线

- 0-3 月：研究、策略开发
- 3-6 月：回测、模拟盘
- 6-12 月：小资金实盘

- **资金不足**：被风险事件击溃

```python
async def low_latency_handler(url: str, callback):

                                   ping_interval=None,
                                   max_size=2**20,
                                   compression=None) as ws:
        recv_t = []
        async for msg in ws:
            t = time.perf_counter_ns()
            data = json.loads(msg)
            await callback(data, t)
            recv_t.append(time.perf_counter_ns() - t)

                print(f"avg callback time: {avg:.2f} ms")
                recv_t.clear()
```

## A.2 订单簿事件流处理

```python
"""
维护 L2 订单簿，处理增量更新
"""
from sortedcontainers import SortedDict


class FastOrderBook:
    def __init__(self):
        self.bids = SortedDict()
        self.asks = SortedDict()
        self.last_update_ts = 0

    def apply_delta(self, delta: dict):
        for px, qty in delta.get('b', []):
            px, qty = float(px), float(qty)
            if qty == 0:
                self.bids.pop(px, None)
            else:
                self.bids[px] = qty
        for px, qty in delta.get('a', []):
            px, qty = float(px), float(qty)
            if qty == 0:
                self.asks.pop(px, None)
            else:
                self.asks[px] = qty
        self.last_update_ts = delta.get('E', 0)

    def best_bid_ask(self) -> tuple:
        bid = self.bids.keys()[-1] if self.bids else 0
        ask = self.asks.keys()[0] if self.asks else float('inf')
        return bid, ask
```

## A.3 Glosten-Milgrom 信息模型

```python
"""
Glosten-Milgrom 1993 做市定价模型
- 做市商不知道订单是信息驱动还是噪声
- 设定先验 V_high / V_low + 到达概率 α
"""
import numpy as np


class GlostenMilgrom:
    def __init__(self, v_low: float, v_high: float, alpha: float):
        self.v_low = v_low
        self.v_high = v_high
        self.alpha = alpha

    def quotes(self) -> dict:
        ask = self.alpha * self.v_high + (1 - self.alpha) * self.v_low
        bid = self.alpha * self.v_low + (1 - self.alpha) * self.v_high
        return {"ask": ask, "bid": bid, "spread": ask - bid}
```

## A.4 延迟测量

```python
"""
测量信号 → 决策 → 下单 → 确认 全链路延迟
"""
import time
from dataclasses import dataclass


@dataclass
class LatencyStats:
    n: int = 0
    total_us: int = 0
    max_us: int = 0

    def record(self, us: int):
        self.n += 1
        self.total_us += us
        self.max_us = max(self.max_us, us)

    @property
    def avg_us(self) -> float:
        return self.total_us / self.n if self.n else 0


def measure_latency(stats: LatencyStats, start_ns: int, end_ns: int):
    stats.record(end_ns - start_ns)
```

## A.5 协程 vs 多进程

```python
"""
asyncio: I/O 密集（WebSocket, REST）
multiprocessing: CPU 密集（信号计算, ML 推理）
"""
import asyncio
import multiprocessing as mp


def cpu_intensive_worker(price_queue: mp.Queue, signal_queue: mp.Queue):
    while True:
        book = price_queue.get()
        if book is None:
            break
        signal_queue.put({"action": "buy", "px": 100.0})


async def main_async():
    price_q = mp.Queue()
    signal_q = mp.Queue()
    pool = mp.Pool(4, cpu_intensive_worker, (price_q, signal_q))
    # ... async 接收行情 + 推送
    pool.close()
    pool.join()
```
