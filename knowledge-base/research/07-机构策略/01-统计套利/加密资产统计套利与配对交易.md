## 一、统计套利基础

### 1. 概念定义

**统计套利（Statistical Arbitrage）**：
- 利用资产价格的**统计关系**进行套利
- 基于量化模型识别**暂时性偏离**
- 期望价格在统计上回归常态

**关键特征**：
- 多空对冲（市场中性）
- 短期持有（秒到天）
- 高频率、高胜率
- 基于历史数据

**与无风险套利的区别**：
- 统计套利是"概率游戏"（不是无风险）
- 期望值为正，但不是确定
- 长期累积微小优势

### 2. 历史渊源

**Pairs Trading 起源**（1980s）：
- Morgan Stanley 的 Nunzio Tartaglia 团队
- 1987 年股灾后开发
- **Gatev, Goetzmann, Rouwenhorst (2006)** 经典论文
- 样本：1962-2002 美国股票
- 半年期配对，年化 11%
- 风险：模型风险、相关性变化

**统计套利发展**：
- Avellaneda & Lee (2010) "Statistical Arbitrage in the US Equities Market"
- 多因子模型
- 风险管理框架
- 高频化

## 二、配对交易（Pairs Trading）

### 1. 基本思想

**配对交易**：
- 找到两个**历史相关**的资产
- 当价差（spread）扩大时，做多弱势方 + 做空强势方
- 价差回归时，**双向盈利**
- **市场中性**：市场涨跌不直接相关

### 2. 经典案例

**加密版配对**：
- **BTC vs ETH**：历史上相关 0.85+
- **ETH vs SOL**：新兴 ETH 替代
- **LTC vs BCH**：比特币分叉家族
- **UNI vs SUSHI**：DEX 竞争
- **LIDO vs RPL**：流动性质押

**Step 1：选择配对**
- 计算相关性
- 计算协整性（cointegration）
- 选择相关性 + 协整性都高的配对

**Step 2：计算价差**
- 价差 = log(P_A) - β × log(P_B)
- β = 协整回归系数

**Step 3：标准化价差（Z-Score）**
- Z = (当前价差 - 历史均值) / 历史标准差

**Step 4：交易信号**
- Z > +2：价差扩大 → 卖 A + 买 B
- Z < -2：价差缩小 → 买 A + 卖 B
- Z 接近 0：平仓

### 3. 协整性测试

**Engle-Granger 协整**：
- 对两资产价格做 OLS 回归
- 检验残差是否平稳（ADF 测试）
- 残差平稳 → 协整

**Johansen 协整**：
- 处理多个资产
- 更稳健

**加密应用**：
- 大多数 BTC 衍生币协整性在 0.5-0.8
- 减半事件附近协整性下降
- 监管事件冲击下协整性短期失效

### 4. 实操案例

**BTC/ETH 配对**（2024 年示例）：

数据：
- BTC 价格 65,000
- ETH 价格 3,500
- 历史协整系数 β = 0.85
- 价差历史均值 μ = 8.0
- 价差历史标准差 σ = 0.5

计算：
- 当前价差 = ln(65000) - 0.85 × ln(3500) = 11.082 - 0.85 × 8.161 = 11.082 - 6.937 = 4.145
- Z = (4.145 - 8.0) / 0.5 = -7.71

判断：Z 极小（实际应该用 BTC/ETH 比率更合适）

**修正**：
- 用价格比率：BTC/ETH = 65000/3500 = 18.57
- 长期均值 19.50，标准差 1.5
- Z = (18.57 - 19.50) / 1.5 = -0.62
- 接近均值，无信号

**修正 2**：
- BTC 跌到 60,000，ETH 跌到 3,000
- BTC/ETH = 60,000/3,000 = 20.00
- Z = (20.00 - 19.50) / 1.5 = 0.33
- 略高估，做空 BTC/ETH 比率（即买 ETH 卖 BTC）

### 5. 配对交易的局限

**共同因素冲击**：
- 减半、FOMC、监管 → BTC 和 ETH 一起跌
- 价差变化不大，但**绝对损失**

**流动性风险**：
- 弱势方可能流动性差
- 退出困难

**协整性失效**：
- 监管、技术变化导致协整性失效
- 历史上有效不代表未来

## 三、统计套利框架（Avellaneda-Stoikov）

### 1. 模型

**核心思想**：
- 任何资产的对数价格：dS = μ dt + σ dW
- 但**短中期**有均值回归倾向
- 用 OU 过程（Ornstein-Uhlenbeck）建模

**OU 过程**：
- dX = κ(μ - X) dt + σ dW
- κ：均值回归速度
- μ：长期均值
- σ：波动率

**s-score**：
- s = (X - μ) / σ_equ
- σ_equ = σ / √(2κ)
- 当 |s| > 1.5 时建仓
- 当 |s| < 0.5 时平仓

### 2. 加密应用

**用 OU 建模 BTC/ETH 价差**：
- 数据：过去 90 天 1h K 线
- 拟合 OU 参数
- 计算 s-score
- 交易信号

**实战调整**：
- 加入动态阈值
- 考虑手续费和滑点
- 加入风控（最大持仓、最大亏损）

## 四、协整套利（Cointegration-based）

### 1. 多资产协整

**选币池**：
- Top 50 加密币种
- 计算两两协整
- 形成"协整网络"

**最佳配对**：
- 选择协整性最高的 N 对
- 同时交易多对
- 分散风险

### 2. PCA 统计套利

**主成分分析（PCA）**：
- 取多个币种的历史价格
- 提取主成分（市场因素）
- 残差（idiosyncratic）作为信号

**逻辑**：
- 加密币种有共同的市场因素（BTC 主导）
- 残差反映**特异**部分
- 残差均值回归 → 套利机会

**实战**：
- 用 Top 20 币种
- 每日重新估计 PCA
- 监控残差 s-score
- 触发阈值交易

### 3. 协整 vs 相关性

| 指标 | 相关性 | 协整 |
|------|--------|------|
| 衡量 | 同时刻同向 | 长期均衡 |
| 范围 | -1 到 +1 | 通过检验有/无 |
| 短期 | 敏感 | 不敏感 |
| 长期 | 不可靠 | 可靠 |
| 配对交易 | 基础 | 高级 |

**结论**：
- 配对交易应基于**协整**
- 相关性高但协整性低 → 不可用
- 协整性高 → 价差有回归倾向

## 五、跨交易所套利

### 1. 跨交易所价差

**原因**：
- 流动性分散（20+ 交易所）
- 资金转移延迟
- 法币入金/出金差异
- 监管差异

**典型价差**：
- BTC：通常 < 0.05%
- 山寨币：可达 0.5-2%
- **韩元溢价**（Kimchi Premium）：2021 年高达 20%

### 2. 三角套利

**定义**：
- 在同一交易所内
- 利用三个交易对的价差
- 闭合三角

**例**：BTC/USDT、ETH/USDT、ETH/BTC
- 1 BTC = 65,000 USDT
- 1 ETH = 3,500 USDT
- 1 ETH = 0.054 BTC
- 应有：0.054 × 65,000 = 3,510 USDT
- 实际：3,500 USDT
- 差异：0.28%
- 套利空间小

**实战**：
- 利润 < 手续费时不做
- 监控多个三角形
- 高频扫描机会

### 3. 跨期套利

**现货 vs 季度期货**：
- 通常基差：年化 5-15%
- 持有至交割，收敛

**现货 vs 永续**：
- Funding Rate 套利
- 详见其他文档

### 4. 跨链套利

**同一币种在不同链**：
- ETH（Ethereum）vs ETH（Layer 2）vs ETH（其他链）
- 跨链桥接成本
- 套利空间小，但规模化可赚

## 七、加密统计套利实战

### 1. 配对交易回测（Python 示例）

```python
import numpy as np
import pandas as pd
from statsmodels.tsa.stattools import coint, adfuller

# 加载数据
btc = pd.read_csv('btc.csv', index_col='date', parse_dates=True)['close']
eth = pd.read_csv('eth.csv', index_col='date', parse_dates=True)['close']

# 1. 协整检验
score, pvalue, _ = coint(btc, eth)
print(f"Cointegration p-value: {pvalue}")
# pvalue < 0.05 → 协整

# 2. 计算价差
beta = np.polyfit(eth, btc, 1)[0]
spread = btc - beta * eth

# 3. Z-Score
z = (spread - spread.mean()) / spread.std()

# 4. 信号
signal = pd.Series(0, index=btc.index)
signal[z > 2] = -1   # 卖 BTC 买 ETH
signal[z < -2] = 1   # 买 BTC 卖 ETH
signal[(z > 0.5) & (z < 0.5)] = 0  # 平仓

# 5. 计算收益
position = signal.shift(1)  # T+1 执行
returns_btc = btc.pct_change()
returns_eth = eth.pct_change()
strategy_returns = position * (returns_btc - beta * returns_eth)
```

### 2. 多配对组合

**步骤**：
1. 选 50 个币种
2. 两两协整检验
3. 选择最强 10-20 对
4. 等权持有
5. 每日再平衡

**预期表现**：
- 年化 15-30%
- 波动率 8-15%
- Sharpe 1.0-1.5
- **低 Beta**：市场中性

### 3. 风险监控

**关键指标**：
- 平均价差偏离度
- 协整性持续性
- 仓位集中度
- 协整性崩溃告警

**退出条件**：
- 协整性 p-value > 0.10
- 价差扩大 > 5σ
- 连续亏损触发

## A.1 协整性检验（Engle-Granger）

```python
"""
Engle-Granger 两步法：
1) OLS: y = α + β x + ε
2) ADF 检验 ε 是否平稳
- p < 0.05 → 协整
"""
import numpy as np
import pandas as pd
import statsmodels.api as sm
from statsmodels.tsa.stattools import adfuller, coint


def engle_granger_test(y: pd.Series, x: pd.Series) -> dict:
    X = sm.add_constant(x)
    model = sm.OLS(y, X).fit()
    alpha, beta = model.params
    spread = y - beta * x - alpha
    adf = adfuller(spread.dropna(), autolag='AIC')
    return {
        'beta': beta, 'alpha': alpha,
        'adf_stat': adf[0], 'p_value': adf[1],
        'is_cointegrated': adf[1] < 0.05
    }


def cointegration_matrix(prices: pd.DataFrame) -> pd.DataFrame:
    cols = prices.columns
    n = len(cols)
    pvals = pd.DataFrame(np.ones((n, n)), index=cols, columns=cols)
    for i in range(n):
        for j in range(i+1, n):
            _, p, _ = coint(prices.iloc[:, i], prices.iloc[:, j])
            pvals.iloc[i, j] = pvals.iloc[j, i] = p
    return pvals
```

```python
"""
Johansen 协整检验（多变量）
- 适合 > 2 个资产的协整关系
"""
import numpy as np
import pandas as pd
from statsmodels.tsa.vector_ar.vecm import coint_johansen


def johansen_test(prices: pd.DataFrame, det_order: int = 0,
                  k_ar_diff: int = 1) -> dict:
    result = coint_johansen(prices, det_order, k_ar_diff)
    return {
        'eigenvalues': result.eig,
        'eigenvectors': result.evec,
        'trace_stat': result.lr1,
        'max_eig_stat': result.lr2,
        'crit_values_trace': result.cvt,
        'crit_values_max_eig': result.cvm
    }
```

## A.3 价差建模（OU 过程）

```python
"""
Ornstein-Uhlenbeck 过程
- dX = κ(μ - X) dt + σ dW
- 半衰期 = ln(2) / κ
"""
import numpy as np
import pandas as pd
from scipy.optimize import minimize


def ou_fit(spread: pd.Series, dt: float = 1.0) -> dict:
    x = spread.dropna().values
    n = len(x) - 1

    def neg_loglik(params):
        kappa, mu, sigma = params
        if kappa <= 0 or sigma <= 0:
            return 1e10
        e = x[1:] - x[:-1] * np.exp(-kappa * dt) - mu * (1 - np.exp(-kappa * dt))
        e_var = sigma**2 * (1 - np.exp(-2 * kappa * dt)) / (2 * kappa)
        return 0.5 * n * np.log(2 * np.pi * e_var) + 0.5 * np.sum(e**2) / e_var

    x0 = [0.1, np.mean(x), np.std(x)]
    res = minimize(neg_loglik, x0, method='Nelder-Mead')
    kappa, mu, sigma = res.x
    return {
        'kappa': kappa, 'mu': mu, 'sigma': sigma,
        'half_life': np.log(2) / kappa,
        'mean_reverting': kappa > 0.01
    }
```

## A.4 配对交易信号

```python
"""
配对交易：
- z = (spread - μ) / σ
- z > +2: spread 偏高 → 做空 spread
- z < -2: spread 偏低 → 做多 spread
"""
import pandas as pd
import numpy as np


def pair_trading_signal(spread: pd.Series, ou_params: dict,
                        entry_z: float = 2.0, exit_z: float = 0.5) -> pd.Series:
    mu = ou_params['mu']
    sigma = ou_params['sigma']
    z = (spread - mu) / sigma
    signal = pd.Series(0, index=spread.index)
    position = 0
    for i, zv in enumerate(z):
        if position == 0:
            if zv > entry_z:
                position = -1
            elif zv < -entry_z:
                position = 1
        else:
            if abs(zv) < exit_z or (position == 1 and zv > entry_z) or \
               (position == -1 and zv < -entry_z):
                position = 0
        signal.iloc[i] = position
    return signal
```

## A.5 跨交易所价差套利

```python
import pandas as pd
import numpy as np


class CrossExchangeArb:
    def __init__(self, exchanges: list[str], fee_bps: float = 10):
        self.exchanges = exchanges
        self.fee = fee_bps / 10000

    def find_arbs(self, prices: pd.DataFrame) -> pd.DataFrame:
        opps = []
        for i, row in prices.iterrows():
            ex_max, px_max = row.idxmax(), row.max()
            ex_min, px_min = row.idxmin(), row.min()
            spread_pct = (px_max - px_min) / px_min
            if spread_pct > 2 * self.fee:
                opps.append({
                    'ts': i, 'buy': ex_min, 'sell': ex_max,
                    'px_buy': px_min, 'px_sell': px_max,
                    'spread_pct': spread_pct,
                    'net_pct': spread_pct - 2 * self.fee
                })
        return pd.DataFrame(opps)
```

```python
"""
永续合约 funding rate 套利
"""
import pandas as pd
import numpy as np


def funding_arb_signal(funding_rate: pd.Series,
                        spot_price: pd.Series,
                        perp_price: pd.Series) -> pd.DataFrame:
    basis = (perp_price - spot_price) / spot_price
    annual_funding = funding_rate * 3 * 365
    signal = pd.DataFrame({
        'funding': funding_rate,
        'basis': basis,
        'annual_funding': annual_funding,
    })
    signal['action'] = np.where(annual_funding > 0.20, 'short_perp_long_spot',
                                np.where(annual_funding < -0.20,
                                         'long_perp_short_spot', 'none'))
    return signal
```
