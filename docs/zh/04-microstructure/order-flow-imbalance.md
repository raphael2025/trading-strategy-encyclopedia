# 订单流失衡(OFI)/ VPIN

> 一句话:从订单簿和成交带测量净买入与净卖出压力,以预判下一步极短视野的价格变动。**判决:🔶 有条件**(优势真实存在,但受众错了)。

## 它是什么

两个相关的微观结构信号。**订单流失衡(order flow imbalance,OFI)** 是盘口顶端的净有向压力:它把买价、卖价挂单量的变化与打到各侧的成交合在一起求和,因此一波买盘挂单的增加加上激进买入会产生一个很大的正 OFI。**VPIN**(volume-synchronized probability of informed trading,成交量同步的知情交易概率)按等量成交量而非时钟时间对成交带分桶,并估计每个桶内买卖成交量之间的失衡——它是 *流动性毒性* 的代理指标,即你正在与某个信息更灵通的对手交易的可能性有多大。两者都读自高频 Level-2 数据,而非K线。

## 主张

价格由订单流推动,因此如果你能在订单簿重新定价 *之前* 测量出失衡,就能预测下一跳。OFI 文献表明,价格变化与净订单簿压力紧密且同期地对齐;VPIN 文献则主张,上升的毒性先于波动率和逆向选择。实践者版本是:"盯住订单流,抢在变动之前下手。"

## 检验

我们在加密永续合约的高频 L2 订单簿数据上评估 OFI,用 **RankIC** 对下一视野收益打分,信号因果对齐打时间戳,然后重新核查在你必须真正 *交易* 它时还剩下什么——吃单/taker 费用、滑点以及你跨越的价差全部内化。

- **亚秒级视野下 RankIC ≈ 0.29。** 这确实很强——远高于把零售K线机器学习和跟单埋葬掉的那种 ≈ 0 的 RankIC。信号是真实的;订单簿确实预告了下一跳。
- **零售吃单者净得 ≈ −8 bps。** 跨越价差、支付吃单/taker 费用、并迟到微秒级,会把全部优势——还要再多搭一些——拱手让给那些早已挂在订单簿上的人。
- 优势衰减不是因为预测错了;它衰减是因为 **捕获它需要你成为流动性,而不是拿走流动性。**

## 判决:🔶 有条件

优势真实,受众错误。预测信息是正当的,且能在诚实打分下存活——这不像 FVG 回踩或K线家族那样是多重检验的人为产物。但这种优势活在 *你必须跨越的价差之内*。只有那些 **提供** 流动性的参与者才能捕获它(赚取挂单/maker 返佣和排队优先权),并凭借同机房、低延迟的基础设施在微秒级行动。对他们而言这是一个可行的策略;对零售吃单者而言,这是每一个往返保证 −8 bps 的亏损。这正是它被判为 🔶 而非 ✅ 的原因:通过 RankIC 检验是必要而非充分的——受众与执行模型决定了优势是真金白银还是费用捐赠。不适合零售。同样的逻辑适用于 [幌骗/吸筹检测(spoofing/absorption detection)](../00-verdict-index.md#microstructure):信息丰富,但在零售延迟下不可执行。

## 自己试试

从 L2 更新重建 OFI,测量它相对下一视野收益的 RankIC——你应当复现出一个很强的、真实的数字。然后把一个在每个信号上跨越价差的吃单者的 *可交易* 收益跑一遍 [`deflate`](https://github.com/raphael2025/deflate),看着净盈亏在成本内化后稳稳坐在负值。两者之间的差距正是全部的教训。如果你想要做市版本,你需要返佣、排队位置和微秒级延迟——那是基础设施,而不是一个笔记本。
