# 01 · 技术指标

本节把最广为传授的图表指标当作**独立的、机械式的买/卖信号**来检验——而当我们诚实地把成本内化、把参数搜索计入之后,它们无一幸免,全部死掉。

| 指标 | 文档 | 判决 | 原因 |
|---|---|---|---|
| RSI | [rsi.md](./rsi.md) | ❌ 已证伪 | 调出来的 30/70 阈值无法泛化;在趋势中去淡化"超买"等于接飞刀;在成本 + 多重检验下死掉。 |
| MACD | [macd.md](./macd.md) | ❌ 已证伪 | (12,26,9) 默认参数是历史遗留产物;交叉信号在震荡中反复打脸;它不过是两条均线之差的滞后量,在成本 + 多重检验下死掉。 |
| 移动平均线 | [moving-averages.md](./moving-averages.md) | ❌ 已证伪 | "用哪一对?"——50/200 是民间俗成挑出来的,样本内最优的那一对无法延续到样本外;在成本 + 多重检验下死掉。 |
| 布林带 | [bollinger-bands.md](./bollinger-bands.md) | ❌ 已证伪 | 20/2 是民间调出来的,而两种相互矛盾的读法让你总能在样本内套上一个;去淡化触带和均值回复一样,都是接飞刀。 |

**关键的元结论:** 独立的、调过参的指标是*对选择视而不见的民间传说*——它们能骗过 bootstrap 和 walk-forward,因为一个精挑细选的配置仍然好看,但它们在 DSR 和 PBO(这两者知道你试过多少个配置)面前崩溃,并且永远过不了现实成本这一关。幸存者用的并不是不同的原料;它们用的是*同样*的原始原料——动量、波动率——只不过被放进了一个**对选择有意识、对成本有意识的系统**里,顺着趋势而不是去淡化趋势。完整地图见[判决索引](../00-verdict-index.md)。
