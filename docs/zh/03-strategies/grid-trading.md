# 网格交易

> 一句话:在当前价格下方挂一排买单、上方挂一排卖单,当价格在一个区间内来回震荡时收割价差。**判决:在趋势中 ❌(趋势会把你碾过去);只有作为区间内、做了库存管理的做市才是 🔶。**

## 它是什么

网格交易在一个价格带上铺一张限价单的网格:每下跌一格买入一份,每上涨一格卖出一份。在横盘市场里,每一对完成的“低买高卖”都记入一小笔利润,机器人不断循环。流行的加密机器人(3Commas、交易所自带的网格机器人)将其自动化,通常带有 `pyramiding`(金字塔)式的累积(同时持有多个仓位),有时还附加一个马丁格尔模块(每进一步下跌就加大仓位)——这会大幅抬高爆仓风险。

## 主张

你不需要预测方向——你只需要一个区间内的波动,网格就会把这种来回震荡变现。它被包装成“一键设好就不用管的被动收入”,尤其是针对横盘的山寨币。

## 检验

这和[流动性提供/均值回复](mean-reversion.md)是同一种动物,在[判决索引](../00-verdict-index.md)中它也得到同样的判决:**在趋势中 ❌——比空仓还差,因为趋势会把你碾过去。**

- **面对趋势时收益结构不对称。** 网格廉价地卖掉它的赢家(在小幅反弹时平掉多单),却*不断累积*它的输家(一路下跌一路买)。在加密那种强趋势的环境里,这意味着你在一轮持续下跌中持有一个越来越大的仓位——带着杠杆去接飞刀。那些小小的区间利润,被那一次击穿价格带的趋势远远盖过。
- **没有方向性边际,而且每一格都有成本。** 每一次成交都要付手续费/滑点;这个策略需要区间震荡得足够多才能覆盖这种换手,而区间不会提前向你宣告自己的存在。你只有事后才知道它曾经是“区间震荡”的([未来函数](../../METHODOLOGY.md))。
- **马丁格尔尾部。** 那个让资金曲线看起来很平滑的附加模块(越跌越买),恰恰就是把一次回撤转化为一次爆仓的元凶。

## 判决:❌(只有作为真正的区间做市才是 🔶)

作为一个零售“被动收入”机器人,网格交易是 ❌:它在平静的区间里盈利,然后在终将到来的趋势中把利润(还要多搭上一些)全部吐回去,同时还在每一格上付成本。*唯一*站得住脚的版本,是在一个真正均值回复的标的上、由专业人士进行的、做了库存管理的做市,带有挂单返佣以及硬性的库存/风险上限——那是 🔶 有条件的,依赖延迟和返佣,且不是零售网格机器人。如果你想要一个真正能存活下来的非方向性加密边际,那是[资金费 carry](../08-crypto-specific/funding-rate-carry.md),而不是价格网格。

## 自己试试

在一个区间窗口和一个趋势窗口上分别回测网格,每次成交都计入成本,然后跑 [`deflate`](https://github.com/raphael2025/deflate) 的 `bootstrap`(看看趋势那一段带来的肥厚左尾)和 `placebo`。只看区间窗口的结果是幸存者偏差;在包含趋势的完整样本上,扣除成本后的收益会变为负值。
