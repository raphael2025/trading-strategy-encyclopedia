# 止损

> 一句话:旨在"砍掉损失、保护本金"的保护性出场规则——固定/百分比止损、过早止盈、在阻力位出场,相对于宽的、基于 ATR 的移动止损。**判决:❌ 紧止损已证伪 / 🔶 宽 ATR 有条件**。

## 它是什么

止损是一个预先承诺的出场。两个非常不同的家族藏在同一个词下面。**紧止损**在一个小的不利变动后平仓(一个固定百分比、支撑下方一跳);它的近亲是**过早止盈**(锁定一笔小收益)和**在阻力位出场**。**宽 ATR 移动止损**远离价格——是平均真实波幅(Average True Range)的某个倍数——只在一个大的、结构性的反转上才触发,并随着交易顺行而向上棘轮式调整。第一个家族封顶任何单次走势的*大小*;第二个家族只捕捉*灾难性*的那些。

## 主张

止损"砍掉损失"因而保护本金。在散户教学中这被当作无条件的好事:更紧的止损被呈现为更安全、更有纪律的交易。

## 检验

我们在已验证的[趋势跟随](../03-strategies/trend-following.md)优势之上应用每一种止损类型——一个只在**约 36% 的时间**取胜、却携带**约 6.5 盈亏比**的系统——信号在 K 线收盘时打时间戳(无未来函数),成本内化。

- **紧止损 / 过早止盈 / 阻力位出场 = ❌。** 它们在那少数巨大赢家*成熟之前*就把它们砍掉。因为全部期望值都活在那条肥厚的右尾里,截断它就把一个正期望系统变成一个被千刀万剐拖死的系统:你保留了 64% 的小亏损,扔掉了为它们买单的那些罕见赢家,还在上面加了换手成本。一旦栓上一个止盈,Sharpe 就崩塌。
- **宽 ATR 移动止损 = 🔶——它有帮助。** 设得足够远,使普通噪声够不着它,它就砍掉*灾难性*的反转(那条剧烈翻转的趋势),同时让右尾完好无损。它是尾部保险,不是利润规则。

## 判决:❌ 紧止损已证伪 / 🔶 宽 ATR 有条件

判决完全依**宽度**而分裂。正确的问题从来不是"止损好不好?",而是"这个止损是否截断了优势所依赖的右尾?"紧止损会,而它们对任何肥尾动量系统都是致命的;致命之处很简单——**它们摧毁了盈亏比**,而盈亏比*就是*优势。宽 ATR 移动止损是有条件的:作为灾难防护,它能挣回自己的成本,前提是它足够宽、正常波动率永远碰不到它。现实的预期:根本不要把止损当作你的首要风险工具——那个干净的、数学上可靠的旋钮是[仓位](position-sizing-and-vol-targeting.md),不是出场择时。还要注意这个陷阱:紧止损*感觉*安全(它们抚慰了[损失厌恶](../05-psychology/cognitive-biases.md)),而恰恰就在此时它们把策略放干了血。

## 自己动手试试

运行已验证的趋势系统,然后加入 (a) 一个紧的百分比止损,(b) 一个过早止盈,以及 (c) 一个宽 ATR 移动止损,成本内化。看着 (a)/(b) 下胜率上升而总收益*下降*,因为右尾被剪掉了,再看着 (c) 在不掏空大赢家的情况下削减最糟糕的回撤。把这与单纯通过[波动目标](position-sizing-and-vol-targeting.md)降低仓位作比较——通常那是更好的交易。
