# 恐惧贪婪指数

> 一句话:一个 0–100 的情绪综合指标,标注为从"极度恐惧"到"极度贪婪",被当作一个逆向择时信号兜售——恐惧时买,贪婪时卖。**判决:❌ 已证伪**(作为买/卖信号)。

## 它是什么

一个单一指数,在加密和股票领域都被普及(CNN / Alternative.me 风格),它把若干输入混合成一个 0 到 100 的数字:近期**波动率**、价格**动量**、**社交/调查情绪**、市场**主导度/广度**,有时还有期权或避险需求。读数被分桶成从*极度恐惧*经*中性*到*极度贪婪*的标签。

## 主张

纯粹的逆向思维:"在别人恐惧时贪婪,在别人贪婪时恐惧。"极度恐惧本应标记投降式的底部(买入),极度贪婪本应标记狂热的顶部(卖出)。该指数被呈现为一个市场择时触发器。

## 检验

我们把极度恐惧和极度贪婪的阈值当作 BTC 和股票上的**入场信号**,读数按因果方式打时间戳(无未来函数),**吃单/taker 费用 + 滑点 + 资金费内化**,把阈值当作一个**搜索网格**(因为"极度"是一个可调的切点),并把结果与匹配的**安慰剂**以及买入持有进行比较。

- **该指数在很大程度上是近期价格和波动率的一个滞后变换**——"贪婪"*就是*近期高收益,"恐惧"*就是*近期回撤加上高波动率。用它来给入场择时因此接近于循环论证:你是在你本想预测的那个走势上做条件。
- 阈值是**可调的**,所以一个样本内的"最佳"切点是必然能找到的;在选择感知的打分下,这种优势会被压缩(deflate)掉。
- **扣除成本后,它既打不赢安慰剂,也打不赢买入持有。** 在下跌趋势中买入"极度恐惧",和那个拖垮[均值回复 / 抄底](../03-strategies/mean-reversion.md)的"接飞刀"问题如出一辙。

## 判决:❌ 已证伪

作为一个独立的、考虑成本的**买/卖信号**,恐惧贪婪指数无法存活:它滞后且基本是循环论证,它的阈值招致多重检验,扣除成本后相比安慰剂或单纯持有毫无增益。致命之处在于,该指数*就是*近期价格/波动率的重新包装——所以作为择时工具,它预测的是过去。

**诚实的正面:** 它作为一个**描述性温度计**完全合格。作为一个风险恒温器——当指标钉在狂热区时减小仓位,在极端处预期脆弱性——它是一个合理的纪律辅助,与[认知偏差](cognitive-biases.md)中描述的那种防御性用法相同,并通过[仓位与波动目标](../06-risk-management/position-sizing-and-vol-targeting.md)落地实施。是温度计,不是触发器。

## 自己动手试试

拉取历史指数,在你选定的极端阈值处定义入场,内化成本,然后把收益跑一遍 [`deflate`](https://github.com/raphael2025/deflate):扫描阈值,让 `deflated_sharpe` 惩罚这个网格,检查 `pbo`,并与 `placebo` 和买入持有比较。然后把指数对滞后收益和已实现波动率做回归——一旦你看到它有多大部分只是*近期价格*,循环论证就显而易见了。
